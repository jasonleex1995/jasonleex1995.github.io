## 프로젝트 개요 (확정된 방향)

- **목표**: NAN 2026 공모전(nan2026.nhn.com) 사전과제인 *AI 활용 게임 제작*에 참가.
- **구현 방식**: 별도 repo 없이, 이 블로그 `projects/nan2026/` 안에 **바닐라 HTML/Canvas/JS**로 self-contained 정적 페이지 구현 (기존 `projects/exif-frame/` 패턴과 동일).
- **이유**:
  - 플레이 가능 — `https://jasonleex1995.github.io/projects/nan2026/`에서 브라우저로 바로 실행.
  - 최소 용량 · no legacy — 프레임워크 런타임·`node_modules`·빌드 스텝 없음. 게임 하나가 수십 KB 수준(갤러리 사진 1장보다 작음).
  - 관리 단순 — repo 하나, `data/projects.json` 한 줄로 등록, 블로그 톤(styles.css·footer) 공유.
- **슬러그**: `nan2026` (공모전명 기반). 아직 projects.json 미등록이라 필요 시 폴더명(=URL) 변경 비용 0.
- **저장소 (확정, 방법 1)**: 별도 repo 없이 블로그 repo 안 `projects/nan2026/`에서 개발 = **단일 원본**. 복붙 동기화 없음. 공모전 5개 제출 요건 모두 충족. 필요 시 **마감 직전** `git subtree split --prefix=projects/nan2026`로 게임만 히스토리째 별도 repo로 추출 가능(옵션, 지금은 안 함).
- **커밋 규칙 (심사용 clean history)**:
  1. 게임 커밋 메시지는 접두어 **`nan2026:`** 사용.
  2. 한 커밋은 가능한 한 **`projects/nan2026/` 파일만** 건드림 (블로그 콘텐츠 커밋과 섞지 않음).
  3. 의미 단위로 자주 커밋해 개발 과정이 이야기처럼 읽히게.
  → 심사자는 `git log --oneline -- projects/nan2026/` 로 게임 개발 과정만 깔끔히 볼 수 있음.
- **게임 종류**: **미정.** Claude와 차근차근 기획 예정. 용량·단순성 고려 시 에셋 없는 도형 기반 게임(스네이크·2048·테트리스·브레이크아웃 등)이 유력 후보.

### 2026-07-13 — AI 활용 방향 확정: "제작 도구로서의 AI" + 무-API 오프라인 게임

- **나**: AI는 게임의 **기획·디자인·개발·디버깅(QA)**에 쓰고 싶다. 게임 안에 AI를 넣으면 API가 필요해 리소스가 크다고 봄. 목표는 웹에서 **저용량·안정적**으로 도는 게임.
- **Claude**: 이해함. **결정 — AI는 제작 프로세스 도구, 게임 자체는 저용량·안정·완전 오프라인·무API.**
  - *정정*: "인게임 AI = 반드시 API"는 아님. TELL·Oracle·Mirror Pong·2048:Adversary의 AI는 **LLM API가 아니라** 브라우저에서 도는 고전 알고리즘(예측 모델·탐색·초소형 신경망) → 몇 KB, 오프라인, 완전 안정. 즉 "저용량·안정" 목표와 충돌하지 않음. (다만 사용자는 심플한 게임을 선호 → 옵션일 뿐, 강요 안 함.) **LLM API 기반 인게임 AI만** 무겁고 불안정하므로 그건 어느 방향이든 배제.
  - *전략 유의*: 공모전이 "AI를 활용하는 사람은 많다"고 못박음 → 단순 게임 + "AI로 코딩함"만으론 차별화가 약함. **AI 제작 과정을 풍부하게(설계 반복·에셋 생성·자동 QA·워크플로) 문서화**해야 경쟁력. 이미 멀티에이전트 기획·이 log.md로 그 방향을 타는 중.
- **결정**: 게임 자체는 **무API·오프라인·저용량** 확정. 구체 게임 선택은 계속 상의.

### 2026-07-13 — 방향 A 확정 (심플 게임 + AI 제작과정) & '오프라인 AI' 오해 해소

- **나**: A가 좋다. 웹에서 '오프라인 AI'가 잘 안 와닿음 — 오프라인이면 모델을 다운받아야 하는 것 아닌가?

### 2026-07-13 — 보스 3분 타이머 · 배속 난이도(스피드런) · 코인 규칙 (DESIGN v0.4)
