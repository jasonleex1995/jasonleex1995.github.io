## 프로젝트 개요 (확정된 방향)

- **목표**: NAN 2026 공모전(nan2026.nhn.com) 사전과제인 *AI 활용 게임 제작*에 참가.
- **구현 방식**: 별도 repo 없이, 이 블로그 `projects/nan2026/` 안에 **바닐라 HTML/Canvas/JS**로 self-contained 정적 페이지 구현 (기존 `projects/exif-frame/` 패턴과 동일).
- **이유**:
  - 플레이 가능 — `https://jasonleex1995.github.io/projects/nan2026/`에서 브라우저로 바로 실행.
  - 최소 용량 · no legacy — 프레임워크 런타임·`node_modules`·빌드 스텝 없음. 게임 하나가 수십 KB 수준(갤러리 사진 1장보다 작음).
  - 관리 단순 — repo 하나, `data/projects.json` 한 줄로 등록, 블로그 톤(styles.css·footer) 공유.
- **슬러그**: `nan2026` (공모전명 기반). 아직 projects.json 미등록이라 필요 시 폴더명(=URL) 변경 비용 0.
- **저장소 (확정, 방법 1)**: 별도 repo 없이 블로그 repo 안 `projects/nan2026/`에서 개발 = **단일 원본**. 복붙 동기화 없음. 공모전 5개 제출 요건 모두 충족. 필요 시 **마감 직전** `git subtree split --prefix=projects/nan2026`로 게임만 히스토리째 별도 repo로 추출 가능(옵션, 지금은 안 함).
- **커밋 규칙 (심사용 clean history)**:
  1. 게임 커밋 메시지는 접두어 **`nan2026:`** 사용.
  2. 한 커밋은 가능한 한 **`projects/nan2026/` 파일만** 건드림 (블로그 콘텐츠 커밋과 섞지 않음).
  3. 의미 단위로 자주 커밋해 개발 과정이 이야기처럼 읽히게.
  → 심사자는 `git log --oneline -- projects/nan2026/` 로 게임 개발 과정만 깔끔히 볼 수 있음.
- **게임 종류**: **미정.** Claude와 차근차근 기획 예정. 용량·단순성 고려 시 에셋 없는 도형 기반 게임(스네이크·2048·테트리스·브레이크아웃 등)이 유력 후보.

### 2026-07-13 — AI 활용 방향 확정: "제작 도구로서의 AI" + 무-API 오프라인 게임

- **나**: AI는 게임의 **기획·디자인·개발·디버깅(QA)**에 쓰고 싶다. 게임 안에 AI를 넣으면 API가 필요해 리소스가 크다고 봄. 목표는 웹에서 **저용량·안정적**으로 도는 게임.
- **Claude**: 이해함. **결정 — AI는 제작 프로세스 도구, 게임 자체는 저용량·안정·완전 오프라인·무API.**
  - *정정*: "인게임 AI = 반드시 API"는 아님. TELL·Oracle·Mirror Pong·2048:Adversary의 AI는 **LLM API가 아니라** 브라우저에서 도는 고전 알고리즘(예측 모델·탐색·초소형 신경망) → 몇 KB, 오프라인, 완전 안정. 즉 "저용량·안정" 목표와 충돌하지 않음. (다만 사용자는 심플한 게임을 선호 → 옵션일 뿐, 강요 안 함.) **LLM API 기반 인게임 AI만** 무겁고 불안정하므로 그건 어느 방향이든 배제.
  - *전략 유의*: 공모전이 "AI를 활용하는 사람은 많다"고 못박음 → 단순 게임 + "AI로 코딩함"만으론 차별화가 약함. **AI 제작 과정을 풍부하게(설계 반복·에셋 생성·자동 QA·워크플로) 문서화**해야 경쟁력. 이미 멀티에이전트 기획·이 log.md로 그 방향을 타는 중.
- **결정**: 게임 자체는 **무API·오프라인·저용량** 확정. 구체 게임 선택은 계속 상의.

### 2026-07-13 — 방향 A 확정 (심플 게임 + AI 제작과정) & '오프라인 AI' 오해 해소

- **나**: A가 좋다. 웹에서 '오프라인 AI'가 잘 안 와닿음 — 오프라인이면 모델을 다운받아야 하는 것 아닌가?

### 2026-07-13 — 취향 정교화: "선택이 만든 양학" + 적당한 숙련 레이어 (탈-클론)

- **나**: 프로파일 다 맞음. 단 (1) 뱀서 아류 느낌이 안 나야 함 — 더 재밌는 요소 필요. (2) 단순 양학보다 **내 올바른 선택으로 결과가 뚜렷하게 보이는 양학**을 좋아함. (3) 너무 양학만 하면 지루 → **적당히 숙련 쌓는 과정**이 즐거움. 예: 보스몹에 약간의 패턴, 비행기 게임처럼 **탄막을 어느 정도 보고 피하기** (단, 리듬/트위치 반응은 여전히 싫음 = 관대하게 읽고 피하는 수준).
- **설계 타깃 (확정 방향)**: survivor-like 베이스 + ㉠ **빌드 정체성** — 선택이 스탯 +%가 아니라 *플레이스타일을 뚜렷이 바꾸는* 시너지(올바른 선택 → 눈에 보이는 지배) ㉡ **적당한 스킬 레이어** — 텔레그래프된 보스 패턴 + 관대한(읽고 피할 수 있는) 탄막 회피 ㉢ 여전히 무-트위치·짧은 런·무API·오프라인.
- **다음**: 이 타깃에 맞는 구체적 차별화 안(탈-클론 훅) 탐색 → 사용자 선택.

### 2026-07-13 — 차별화 훅 탐색 (Claude 멀티에이전트: 5각도 15안 → 심사)

- **추천 1위: REFRACT (탄환 포식자)** — 핵심 동사 = 회피가 아니라 **HARVEST(흡수)**. 정면 흡수 콘으로 적/보스 탄막을 *먹고*, 드래프트한 **변환 노드**가 흡수한 탄을 무엇으로 바꿀지 결정:
  - **Nova**(모아서 화면청소 역폭발) / **Swarm**(먹은 탄을 유도탄으로 재발사) / **Bulwark**(먹은 탄을 방어막으로).
  - 선택-지배 읽힘: *"내가 Nova 골라서, 보스의 궁극기가 내 궁극기가 됐다."*
  - 왜 맞음: ㉠ **탈-클론**(탄으로 걸어들어가 먹는 건 뱀서와 구조적으로 다름) ㉡ **선택이 플레이스타일을 통째로 바꿈**(+% 아님), 보스의 가장 무서운 순간에 페이오프 ㉢ **관대한 스킬**(흡수는 순수 이득 → 잘못 읽어도 그냥 평범한 관대한 회피, 트위치 아님). 스킬 = 어느 호를 쓸어담을지 읽기 + 언제 방출할지(내 타이밍).
  - 루프: 18분, 미니보스 ~3분/보스 ~4-5분(총 4-5회)+18분 에이펙스. 관대함 노브(작은 판정·넓은 콘·반응창 ≥500ms·텔레그래프·'BIG ABSORB' 표시). 방출=내 미터 소비(강제 QTE 아님). 보스1(겨우 먹음)→보스3(빌드 클릭, 한 방에 HP 절반)→에이펙스(궁 유도→자기 탄으로 삭제=양학).

- **런너업**: **USURP**(보스 공격을 이겨서 *훔쳐* 내 무기로 — AI 스토리·선택가독성 최강, 조합 폭발 리스크) / **BREAKPOINT**(올바른 빌드로 보스 공격을 *취소/무력화* — 가장 안전, "무서운 탄막이 아예 안 나옴"=가장 순수한 선택-양학).

### 2026-07-13 — 이동 설계 확정 (뱀서 노가다 제거)

- **나**: 뱀서에서 귀찮았던 건 1+2+3 복합. 단, 탄막 피하며 아이템 먹는 *타이밍 잡는 스킬은 원함* (= 4번 "안 움직이고 싶다"는 아님). 문제는 맵 저 멀리 아이템 주우러 가는 무의미한 노가다가 *중요한 데 집중 못 하게* 만드는 것.
- **결정 (REFRACT에 반영할 이동 규칙)**:
  1. **바운드된 단일 화면 아레나** — 스크롤 맵 횡단 제거(2번).
  2. **자동 자석 수집** — 모트가 플레이어에게 날아오거나 처치 시 자동 수집 → 먼 아이템 줍기 노가다 제거(3번).
  3. **이동은 의미 있는 것에만** — 탄막 회피 + 흡수 콘 포지셔닝 + 먹는 타이밍(= 원하는 "적당한 숙련"). 무의미한 도망 제거(1번).
  → 이동은 유지하되 *알짜만* 남김. 뱀서의 아쉬움을 정확히 겨냥해 제거.

### 2026-07-13 — 피드백: 능동 공격 선호 + "얻어낸 양학"(밸런스 중요) → REFRACT 재검토

- **나**: REFRACT = 총알 막아 기 모아 공격, 으로 이해(맞음). 그런데 (1) 너무 단순·재미가 커 보이지 않음. **쉽게 양학하면 아무도 안 함** → 약간의 허들 + 재밌는 구간을 잘 잡는 밸런스가 핵심. (2) 총알 막기가 **너무 수동적** → **내가 능동적으로 공격**하는 게 더 좋음.
- **방향 전환 (확정)**:
  - 핵심 동사: **수동적 흡수 → 능동 공격.** (REFRACT 흡수는 코어에서 내림; 살리면 선택적 서브 요소로만.)
  - 난이도 철학: **"쉬운 양학"은 노잼 → 양학은 허들을 넘어 *얻어내는* 것.** 약간 빡세지만 넘을 만한 곡선 = 재미의 핵심. 이 곡선 튜닝이 곧 AI 시뮬 밸런싱의 역할 → **밸런스를 차별점으로.**
  - 유지: 선택이 만든 지배(빌드) + 적당한 관대 회피 + 바운드 아레나 + 무-노가다.
