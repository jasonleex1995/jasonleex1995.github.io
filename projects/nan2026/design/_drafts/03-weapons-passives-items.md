## 무기 · 패시브 · 아이템 구체화

> 이 절은 **로스터와 규칙을 확정**한다. 확정 이후 밸런싱은 `data/*.json`의 **숫자만** 만진다.
> 원칙: **거동(behavior)은 코드, 수치(number)는 JSON.** 코드에 없는 훅을 요구하는 콘텐츠는 생성 금지.

---

### 0. 스코프 결정 3개 (먼저 못박음)

| 질문 | 확정 | 이유 |
|---|---|---|
| 패밀리 = 무기 몇 종? | **1 패밀리 = 1 무기. 총 12종.** | 세분화하면 "스탯만 다른 무기"가 생겨 특색 원칙(설계 1순위) 위반. 4칸/12종 = 런마다 1/3만 봄 → 다양성 이미 충분. |
| 해금제? | **없음. 12종 전부 런 시작부터 드래프트 풀.** (log의 "보스 처치 시 테마 무기 해금" 옵션 → **폐기**) | 해금제는 드래프트 확률·시뮬 전제·UI를 전부 복잡하게 만듦. 4주 예산 밖. |
| 패시브 슬롯 | **6칸 확정** (5~6 → 6). | 화면을 어지럽히지 않음 + 아래 성장 예산 산술이 6에서 성립. |

### 1. 무기 데이터 모델 ★ (밸런싱=숫자만 의 핵심)

**12 패밀리 = 12개의 손으로 쓴 update 함수.** JSON은 **선언된 파라미터 어휘** 안의 숫자 행(row)만 담는다.

#### 1-1. 파라미터 어휘 (닫힌 목록 — 이 밖의 필드 추가 = 설계 변경)

```
damage, cooldown_ticks, count, spread_deg, speed, pierce, lifetime_ticks,
radius, tick_interval_ticks, knockback,
homing_turn_rate_deg, retarget_interval_ticks, acquire_radius,
orbit_radius, orbit_speed_deg, return_range, bounce_count,
telegraph_ticks, bullet_clear_radius, target_rule
```

`target_rule` enum: `forward` | `nearest` | `densest` | `random_in_arena` | `player_attached`

#### 1-2. ★ 레벨 규칙 (이 한 줄이 프로젝트 목표를 지킴)

> **Lv1~7의 레벨업은 위 어휘의 숫자만 바꾼다. 새 거동은 오직 Lv8 진화에서만 생긴다.**

- Lv1~8은 **공식이 아니라 8행의 전체 파라미터 행 배열**(`per_level[0..7]`)로 명시 → 시뮬이 특정 레벨만 핀포인트 튜닝 가능.
- `pierce: 0→1`, `count: 1→3` 같은 거동성 변화도 **어휘 안**이므로 숫자 튜닝에 해당.
- 진화는 **12개의 저작된 단발 스텝** (설계 노동 12회로 상한 고정).

#### 1-3. 진화 트리거 ★ (확정)

> **진화 = Lv7 → Lv8 레벨업 그 자체.** 자동도 아니고, 별도 카테고리도 아니고, 패시브 페어링도 아니다.

| 채택 안 | 이유 |
|---|---|
| ~~Lv8 도달 시 자동~~ | 선택의 순간이 사라짐. |
| ~~뱀서식 패시브 페어링 필요~~ | **폐기.** 패시브 풀이 무기 설계에 종속 → 순환 의존. 페어 패시브가 안 뜨면 **막다른 운** = "정보 기반 선택 > 운빨" 기둥 정면 위반. 12무기×페어 저작 = 4주 예산 붕괴. |
| ~~드래프트 5번째 카테고리~~ | 카테고리 4개 락 위반. |
| **Lv8 레벨업 카드 = 진화 카드** ✅ | 새 카테고리 0개, 새 RNG 의존 0개. 7번의 픽으로 **얻어낸** 보상 + 카드가 진화 폼을 미리 보여줌(정보 기반 선택). |

- 진화 카드는 시각적으로 구분(`weapon_level` 카테고리 내 `is_evolution: true`).
- **불가역.** 진화체는 **같은 슬롯 인덱스를 그대로 유지** → 슬롯 순서·임뷰 계산에 특별 취급 없음.
- 진화체도 **스탠스 임뷰를 동일하게 받는다.** 진화 전용 속성 규칙 없음.
- 레벨은 Lv8에서 종료. Lv9 이상 없음 → `weapon_level` 풀에서 제외됨.

#### 1-4. 조준 · 슬롯 배치 · 속성 귀속

- **보스 부위는 각각 독립 타겟 엔티티.** `nearest` 계열은 가장 가까운 부위를 노림 → **플레이어가 위치로 부위를 고른다**(자동 타겟이 퍼즐을 대신 풀지 않음).
- **새 무기 획득 = 빈 슬롯 뒤에 추가**(`slot_assign: "append"`). 획득 순서 = 초기 임뷰 우선순위. 드래프트 카드에 `→ 슬롯 3` 표기 필수.
- **속성 귀속 규칙 (3줄, 12패밀리 전부 커버)** — `element_bind` 필드로 데이터화:

| 종류 | `element_bind` | 해당 |
|---|---|---|
| 투사체(탄) | `spawn` — **발사 시점 스탠스로 고정** | BLITZ, FANOUT, SEEKER, LANCE, RETURN, BARRAGE, REARGUARD, OPTION, NOVA |
| 필드 설치물 | `spawn` — **설치 시점으로 고정** | MINE |
| 플레이어 부착 지속체 | `live` — **매 틱 현재 스탠스 반영** | ORBIT, PULSE |

> 왜: `spawn` 고정이 **"불 기뢰 대량 설치 → 물 전환" 악용**을 원천 차단. `live`는 내 기체의 일부(오빗/오라)라 직관적이고, 이미 화면에 붙어 있어 전환이 **즉시 보임**.

### 2. 무기 로스터 (12종 확정)

**시작 무기 = `blitz` 고정 Lv1, 나머지 3칸 비어서 시작.** (선택 화면 없음 = 스코프 + 모두 같은 출발점 = 온보딩. 벌컨이 가장 읽기 쉬운 거동.)

| # | ID / 이름 | 패밀리 | Lv1 발사·형태 | 레벨업이 움직이는 것 | **Lv8 진화 — 무엇이 변하나** |
|---|---|---|---|---|---|
| 1 | `blitz` **블리츠** | 포워드/벌컨 | 정면 1발 / 8틱 / 탄속 9 | `count 1→3`, `damage`, `cooldown`, `spread_deg 0→10` | **오버드라이브** — 연속 발사 시 3초에 걸쳐 `cooldown` -40%까지 **램프업**(피격 시 리셋) + `pierce +1`. 코앞에서 물고 늘어질수록 강해짐. |
| 2 | `fanout` **팬아웃** | 확산/팬 | 정면 3발 부채꼴 40° / 40틱 | `count 3→7`, `spread_deg 40→70`, `damage` | **플레어 팬** — `spread 180°`(반원, 측면 완전 제압) + 각 탄이 사거리 끝에서 `radius 40` 폭발. |
| 3 | `seeker` **시커** | 유도/시커 | 1발 / 45틱 / 최근접 추적 / 선회 4°틱 | `count 1→3`, `homing_turn_rate`, `damage`, `cooldown` | **스웜** — `count ×2` + 각 탄이 **서로 다른 타겟 우선**(중복 시 차선) + 타겟 사망 시 인접으로 1회 재유도. |
| 4 | `lance` **랜스** | 관통 랜스/빔 | 정면 좁은 관통선 1발 / 70틱 / `pierce 3` | `damage`, `pierce 3→8`, `cooldown` | **레일건** — 25틱 차지 텔레그래프 후 **화면 세로 전체** 관통, `pierce ∞`, `cooldown ×1.6`. 줄 세우기의 완성형·대보스 저격. |
| 5 | `orbit` **오빗** | 오빗/퍼리미터 | 공전체 2 / `orbit_radius 60` / 90°초 / 재히트 쿨 20틱 | `count 2→5`, `orbit_radius`, `orbit_speed_deg`, `damage` | **이지스** — 공전체가 **닿은 적 탄을 소거**(개당 60틱 쿨) + `count +1`. 파고들어 몸빵의 캡스톤. |
| 6 | `pulse` **펄스필드** | 오라/필드 | 자기중심 `radius 70` / 30틱마다 틱딜 | `radius`, `damage`, `tick_interval 30→18` | **싱귤래리티** — 반경 내 적을 중심으로 **약한 인력**(보스·엘리트 제외) + `tick_interval ÷2`. 위기 세션 학살. |
| 7 | `mine` **마인필드** | 설치존/기뢰 | 후방 기뢰 1 / 60틱 / 3초 후 or 접촉 시 `radius 50` 폭발 | `count`, `radius`, `cooldown`, `damage` | **클러스터 마인** — 폭발 시 자탄 6개 산개 → 각 `radius 30` 2차 폭발. 흐름 예측의 페이오프. |
| 8 | `return` **리턴** | 부메랑/리턴 | 전방 1발 / 55틱 / 사거리 끝 귀환 / 왕복 2히트 / `pierce ∞` | `count 1→3`, `speed`, `damage`, `return_range` | **체인 리턴** — 귀환 경로가 최근접 적을 **3회 경유**(지그재그), 경유마다 히트. |

| 12 | `nova` **노바** | 노바/화면청소 | 300틱(5초)마다 자기중심 `radius 120` 대형 폭발 | `radius`, `damage`, `cooldown 300→200` | **슈퍼노바** — 2단 폭발(중심 → 확산 링 `radius 260`) + **폭발 반경 내 적 탄 소거**. 위기 세션의 답. |

> `nova` 진화의 탄 소거는 **반경 한정**이라 폭탄(전화면)과 정체성이 겹치지 않음.

### 3. 패시브 (12종 확정 / 6칸 / Lv1~5)

- **패시브 카드 = 획득 또는 레벨업** (같은 `passive` 카테고리. 5번째 카테고리 만들지 않음).
- `passive_max_level = 5`. **레벨 곡선은 공식이 아니라 5개 값 배열**(`values`) → 시뮬이 레벨 하나만 튜닝 가능.
- 각 패시브 = **엔진 훅 정확히 1개.** 아래 훅 목록 = 닫힌 목록. AI 생성 시 이 밖의 효과 금지.

| # | ID / 이름 | 훅 (하나) | 하는 일 | Lv1→5 곡선(초안, `values`) | 성격 |
|---|---|---|---|---|---|
| 1 | `overclock` **오버클럭** | `cooldown_mult` | 전 무기 발사 간격 ↓ | -6 / -11 / -15 / -19 / -22 % | 화력 |
| 2 | `warhead` **탄두 증량** | `damage_mult` | 전 무기 데미지 ↑ | +8 / +15 / +21 / +26 / +30 % | 화력 |
| 3 | `coil` **확장 코일** | `area_mult` (`radius`·`spread_deg`·`orbit_radius`) | 범위계 파라미터 ↑ | +10 / +18 / +25 / +31 / +36 % | 화력(범위 빌드) |
| 4 | `coating` **관통 코팅** | `pierce_add` | 전 무기 `pierce` +N | +1 / +1 / +2 / +2 / +3 | 거동 변형 |
| 5 | `autoload` **다중 장전** | `count_add` | 전 무기 `count` +N | +0 / +1 / +1 / +1 / +2 | 거동 변형 |
| 6 | `resonance` **상성 증폭** ★ | `super_effective_mult` | **×2 상성 배율 자체를 올림** (×0.5·×1은 불변) | ×2.1 / 2.2 / 2.3 / 2.4 / **×2.5** | **속성축 페이오프** |
| 7 | `afterimage` **잔광** | `untargetable_ticks_on_hit` | 피격 후 N초간 **적 조준·유도 타겟에서 제외**(데미지 무효 아님) | 1.0 / 1.4 / 1.8 / 2.2 / 2.6 초 | 생존 룰 |
| 8 | `reactive` **반응 장갑** | `hit_bullet_clear_radius` | 피격 순간 반경 R 내 **적 탄 소거** | 60 / 90 / 120 / 150 / 180 | 생존 룰 |
| 9 | `bulkhead` **강화 격벽** | `max_hp_add` | 최대 HP ↑ | +1 / +1 / +2 / +2 / +3 | 생존 스탯 |
| 10 | `frame` **경량 프레임** | `move_speed_mult` | 이동속도 ↑ | +6 / +11 / +15 / +18 / +20 % | 생존 스탯 |
| 11 | `study` **학습 회로** | `xp_mult` | XP 획득 ↑ | +10 / +18 / +25 / +31 / +36 % | 경제 |
| 12 | `salvage` **노획 프로토콜** | `coin_mult` | 코인 드랍량 ↑ | +12 / +22 / +30 / +37 / +43 % | 경제 |

- 구성 의도: **화력 3 / 거동 변형 2 / 속성 페이오프 1 / 생존 4 / 경제 2** — "+% 만 있는 패시브 풀"을 피하면서(4·6·7·8이 규칙을 바꿈) 저작 비용은 12개 훅으로 상한 고정.
- **`resonance`가 패시브 풀의 앵커**: 속성에 투자한 사람만 값이 나옴 → 패시브 선택이 속성 축과 실제로 커플링되는 유일한 지점(= "선택이 만든 양학"이 패시브에서도 보임).
- **패시브 진화 없음.** (무기 진화만 = 클라이맥스 집중 + 스코프.)

**`data/passives.json` 스키마**
```json
{ "schema_version": 1,
  "passive_max_level": 5,
  "passives": [{ "id": "overclock", "name_ko": "오버클럭",
                 "hook": "cooldown_mult", "values": [-0.06,-0.11,-0.15,-0.19,-0.22] }]}
```

### 4. 성장 예산 (레벨업 50~60회와의 대차대조표) ★

| 투자처 | 픽 수 | 계산 |
|---|---|---|
| 새 무기 | **3** | 시작 무기 1 지급 → 4칸 중 3칸 |
| 무기 레벨 | **28** | 4무기 × Lv1→8 = 7픽 × 4 (Lv8픽 = 진화) |
| 속성 레벨 | **8** | 합계 상한 8 (아래 §4-1) |
| 패시브 | **30** | 6칸 × Lv5 (획득 픽 포함) |
| **총 싱크** | **69** | |
| **런 레벨업** | **~54** (`levelups_per_run_target: 54`, 범위 50~60) | **충족률 ≈ 78%** |

→ **전부 못 찍는다**가 산술적으로 성립. 동시에 후반 풀이 마르지 않음(§6-4 폴백이 잔여 안전망).

#### 4-1. 속성 투자 캡 확정 ★

| 키 | 값 | 이유 |
|---|---|---|
| `element_level_cap_per` | **4** | 무기 슬롯 수와 일치 = "전환 시 4개 전부 부여" |
| `element_level_cap_total` | **8** | **속성별 4만 있으면 불4·물4·풀4 = 항상 최적 매칭 = 문서가 막으려던 붕괴가 확정.** 합계 8이면 **3속성 전부 만렙 불가** → 4/4/0, 4/2/2, 3/3/2 같은 **진짜 다이얼**이 생김. 4/4/0 빌드는 풀 테마 스테이지를 전부 무속성(×1)으로 싸운다 = 설계된 대가. |
| `element_investable` | `["fire","water","grass"]` | **노말은 투자축이 아님** → `노말 +1` 카드는 존재하지 않음. (Q키 자체의 정의는 §속성 시스템 절 소관.) |
| `element_level_offer_requires_weapon_count` | `true` | **현재 무기 수를 초과하는 속성 레벨은 제시하지 않음.** 무기 2개인데 `불+3`이 뜨는 **죽은 투자**를 원천 차단(= 정보 기반 선택 신뢰도). |

### 5. 상점 항목 최종 + 가격 곡선

#### 5-1. 상점 운영 규칙 (확정)

| 항목 | 확정 | 이유 |
|---|---|---|
| 진열 | **전체 목록 상시. 랜덤 진열 없음. 재고 제한 없음(`max_purchases`만).** | 상점 항목 간 **시너지가 0**이라 조합 최적화가 발생할 수 없음 → 체스식 최적화 위험 없음. 유일한 결정은 *"코인=점수를 포기하고 살까"*. 그리고 **운 완화 밸브(리롤)가 랜덤 진열 뒤에 있으면 자기모순**. |
| 시간 제한 | **없음** | 무-트위치 기둥. |
| 열리는 시점 | **스테이지 사이에만** (정확한 순서는 §스테이지 간 전환 시퀀스 소관) | |
| 표시 필수 | 현재가 · **다음 구매가** · 보유 코인 · **"남은 코인 = N점" 환산 실시간 표시** | 환산값이 안 보이면 "아껴서 점수 vs 써서 생존" 긴장이 플레이어에게 **존재하지 않음**. |
| 가격 누적 | **항목별 독립 / 런 전체 누적 (스테이지 리셋 없음)** | |

**가격 공식 (전 항목 공통, 이것만 코드에 있음)**
```
price(item, n) = ceil(item.base_price * item.growth ^ n)     // n = 그 항목의 런 내 구매 횟수
```

#### 5-2. 항목 (11종 확정)

| 분류 | 항목 (`id`) | 효과 | `base_price` | `growth` | `max_purchases` |
|---|---|---|---|---|---|
| **선택권** | `reroll` — 리롤(스톡) | 드래프트 1회 재추첨 | 30 | 1.60 | 6 |
| **생존/보험** | `potion` — 회복 물약(즉시) | HP `potion_heal_pct` = 35% 회복 | 40 | 1.50 | 8 |
| | `bomb` — 폭탄 스톡 | 스톡 +1 | 50 | 1.35 | 5 |
| | `shield` — 실드(스톡) | 피격 1회 무효 | 60 | 1.50 | 3 |
| | `time_token` — 시간 연장 토큰(스톡) | 보스 타이머 **+`time_token_seconds` = 30 게임초** | 80 | 1.80 | 2 |
| | `continue` — 컨티뉴 | 런당 1회 하드캡 | 250 | — | **1** |
| **비-빌드 유틸** | `defense` — 방어력 ↑ | `damage_taken_mult` ×0.92 (정률, 곱연산) | 45 | 1.45 | 5 |
| | `maxhp` — 최대 HP ↑ | 최대 HP +1 (소폭) | 55 | 1.50 | 4 |
| | `movespeed` — 이동 속도 ↑ | +5% | 40 | 1.50 | 4 |
| | `magnet` — 수집 반경 ↑ | +20% | 35 | 1.40 | 4 |
| | `resist` — 상태이상 저항 ↑ | 둔화·스턴 **지속시간 -20%** | 40 | 1.40 | 3 |
| **폐기** | ~~밴시~~ / ~~무적시간↑~~ / ~~무기·속성 레벨~~ | — | | | |

**해소한 모순 2개**
- ~~"상태이상 저항/**해제**"~~ → **저항만 채택.** '해제'는 발동 키가 필요한데 양손(방향키+QWER)이 이미 만석 → 조작 설계 변경을 유발. 저항은 영구 정률이라 키가 필요 없음.
- **모든 스탯 항목에 `max_purchases` 부여** → 무한 스택 이동속도/방어력으로 **코인이 사실상 빌드를 사는** 우회 차단(대원칙 방어).

**스톡 상한 (구매 상한과 별개)**
```json
{ "bomb_stock_start": 1, "bomb_stock_max": 3,
  "shield_stock_max": 3, "time_token_stock_max": 2, "reroll_stock_max": 5 }
```
- **실드는 순차 소모**(1피격 = 1개). 동시 다중 적용 없음.

#### 5-3. 리롤 규칙 (밴시 폐기 후 유일한 운 완화 밸브 — 확정)

| 규칙 | 값 |
|---|---|
| 구매 시점 / 사용 시점 | **상점에서 스톡 구매 → 드래프트에서 소비** (`reroll` 스톡, HUD 표기) |
| 대상 | **3장 전체 재추첨** |
| 이전 카드 재등장 | **금지** — 방금 본 3장은 그 드래프트 동안 풀에서 제외 |
| 한 드래프트당 연속 사용 | `reroll_max_per_draft: 2` |
| 가격 상승 리셋 | **없음**(런 전체 누적) |

> 왜 "전체 재추첨 + 이전 카드 제외"인가: 리롤이 **재도박**이면 운 완화가 아니다. 이전 3장을 제외해야 리롤이 **확정적인 개선**이 되어 "정보 기반 선택 > 운빨" 기둥을 실제로 지탱한다. 대신 `2회/드래프트` 상한으로 "풀 전체 검색"은 차단.

### 6. 드래프트 규칙 ★

#### 6-1. 뽑는 방식

카테고리를 먼저 뽑지 않는다. **유효 후보 아이템 전체를 가중치 목록으로 만들고 비복원 3장 추첨.**

```
weight(item) = draft_weights_base[item.category] * modifier(item)
```

```json
{ "draft_weights_base": { "new_weapon": 20, "weapon_level": 40, "element_level": 20, "passive": 20 },
  "new_weapon_slot_scale":       [3.0, 1.5, 1.0],   // 현재 무기 수 1 / 2 / 3 일 때
  "weapon_level_evolution_bonus": 2.0,             // Lv7→Lv8(진화) 카드
  "element_first_level_bonus":    1.5,             // 해당 속성 레벨 0일 때
  "passive_new_bonus":            1.3              // 미보유 패시브 & 슬롯 여유
}
```

- `weapon_level`이 40으로 가장 큼 = 28픽 싱크가 가장 크므로 정합.
- 가중치는 **슬롯 상황에 따라 동적**(고정 비율 아님) → 초반엔 무기가 잘 뜨고, 만석 후 자동으로 다른 축으로 흐름.

#### 6-2. 유효성 필터 (무효 카드는 **애초에 풀에 없음**)

| 상황 | 처리 |
|---|---|
| **무기 4칸 만석** | `new_weapon` **카테고리 전체를 풀에서 제외. 교체 제안 없음.** |
| 무기가 Lv8(진화 완료) | 그 무기의 `weapon_level` 카드 제외 |
| 패시브 6칸 만석 | 미보유 패시브 카드 제외 (보유 패시브 레벨업만 남음) |
| 패시브 Lv5 | 그 패시브 카드 제외 |
| 속성이 `cap_per`(4) 도달 | 그 속성 카드 제외 |
| 속성 합계가 `cap_total`(8) 도달 | **3속성 카드 전부 제외** |
| 속성 레벨 ≥ 현재 무기 수 | 그 속성 카드 제외 (죽은 투자 차단) |
| **중복** | **한 드래프트의 3장은 서로 다른 아이템.** 카테고리 중복은 허용(예: 무기 레벨업 2장 + 패시브 1장). |

> **무기 교체를 거부하는 이유 (확정):** ① 투자한 레벨·진화가 증발 = 게임 최악의 feel-bad ② 교체가 **슬롯 순서를 조용히 바꿔 임뷰가 바뀜** — 무관한 선택에서 핵심 시스템이 몰래 변함 = 정보 기반 선택 배신 ③ "버릴 것 고르기" UI + 확인 플로우 = 스코프. 4칸이 차기 전에는 `new_weapon_slot_scale`이 획득을 강하게 밀어주므로 **만석 도달은 정상 진행이지 손해가 아님**.

#### 6-3. 보장 규칙 (3장이 다 쓸모없는 상황 금지)

| 키 | 값 | 하는 일 |
|---|---|---|
| `guarantee_element_card_on_first_draft` | `true` | **첫 드래프트(Lv2)에 속성 카드 1장 확정** → 시그니처 시스템을 반드시 만남. Lv1 전원 속성 0이라 QWER이 no-op인 구간을 최소화(온보딩). |
| `guarantee_new_weapon_until_slots` | `2` | 무기 수 < 2면 `new_weapon` 카드 1장 보장 → 런이 확실히 굴러감. |
| `element_card_pity` | `6` | 최근 6회 드래프트에 속성 카드가 0장이었고 투자 여지가 있으면 **다음 드래프트 1장을 속성 카드로 강제**. 속성축이 운으로 통째 봉인되는 것 차단. |
| `distinct_items_per_draft` | `true` | 3장 서로 다른 아이템. |

#### 6-4. 풀 고갈 폴백

유효 후보가 3장 미만이면 부족분을 **`resupply` 카드**로 채운다.

```json
{ "fallback_card": { "id": "resupply", "name_ko": "보급", "coins": 40 } }
```
- 코인 +40. **절대 무효가 아님**(남은 코인 = 점수 환산이므로 최소한 점수는 됨).
- 빈 드래프트 화면이 물리적으로 불가능해짐 = 구현 안전망.
- 성장 예산상 정상 런에서는 **거의 안 뜬다**(69 싱크 vs 54 레벨업). 뜬다면 그건 곧 "다 찍었다"는 뜻이라 페이오프 구간의 보상으로도 읽힘.

#### 6-5. 카드가 반드시 표시하는 것 (정보 기반 선택 기둥)

| 카테고리 | 카드 필수 표기 |
|---|---|
| `new_weapon` | 이름 · 거동 1줄 · **`→ 슬롯 N에 배치`** · 현재 스탠스에서 **임뷰되는지 여부** |
| `weapon_level` | `Lv3 → Lv4` · **변하는 파라미터의 before → after** · 진화 카드면 **진화 폼 프리뷰** |
| `element_level` | `불 +1 (2 → 3)` · **`전환 시 부여 무기: 2개 → 3개`** ← 이 한 줄이 슬롯 순서 규칙을 가르치는 유일한 지점 |
| `passive` | 이름 · 훅 1줄 · `Lv2 → Lv3` 수치 before → after |

  "levelups_per_run_target": 54,
  "weapon_slots": 4, "passive_slots": 6,

  "element_investable": ["fire","water","grass"],
  "element_level_offer_requires_weapon_count": true,
  "starting_weapon": "blitz",
  "slot_assign": "append",
  "draft_size": 3,
  "draft_weights_base": { "new_weapon": 20, "weapon_level": 40, "element_level": 20, "passive": 20 },
