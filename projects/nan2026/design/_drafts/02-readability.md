## 가독성 · 비주얼 언어 ★최우선★

> 이 섹션은 **기둥 3개(정보 기반 선택 · 적당한 관대 숙련 · 선택이 만든 양학)의 유일한 전달 매체**다. 시스템이 작동해도 여기서 읽히지 않으면 게임은 존재하지 않는다.
> 아래 모든 수치는 `data/visual.json` 키다. **밸런싱 = 이 키의 값만 변경.** 규칙(색 역할·레이어·불변식)은 값이 아니라 계약이므로 변경 금지.
> **소관 경계**: 캔버스 규격·HUD 배치는 [화면·아레나] 섹션, BGM/믹스/에셋 방침은 [오디오] 섹션, 속성 귀속 메커닉은 [속성 시스템] 섹션. 이 섹션은 **"무엇이 어떻게 보이고 들리는가"만** 확정한다.

### 0. 불변식 4개 (Invariants — 코드 리뷰 체크리스트)

| # | 불변식 | 의미 |
|---|---|---|
| **I-1** | **테두리가 있으면 나를 아프게 한다** | 검은 하드 외곽선(`palette.threat.outline`)은 **적 탄·텔레그래프 전용**. 플레이어 탄·FX·픽업·오라는 외곽선 **금지**. |
| **I-2** | **색은 거짓말하지 않는다 (No Lying Color)** | 화면에 그려진 속성색 = 그 개체가 **지금 적에게 적용할 배율**. 발사 후 색이 바뀌면 실제 배율도 바뀐 것이어야 하고, 그 반대도 참. 예외 없음. |
| **I-3** | **텔레그래프 없이는 발사 없다** | 텔레그래프 예산(`caps.telegraphsVisible`)이 곧 공격 예산. 예산 초과 시 **공격을 연기**한다. "텔레그래프 생략하고 발사"는 어떤 상황에서도 금지. |
| **I-4** | **적 탄은 절대 가려지지 않는다** | 적 탄은 항상 최상단 게임플레이 레이어, 불투명, additive 금지. 플레이어의 어떤 이펙트도 적 탄을 덮을 수 없다(레이어 구조가 물리적으로 보장). |

> I-1·I-4가 **무기 4칸 상한의 실제 집행 장치**다. "6칸이면 화면이 죽는다"는 슬롯 수 문제가 아니라 합성 규칙 문제였고, `render.playerFxCompositeAlpha`가 그것을 하드캡한다(§9).

### 1. 색 · 글리프 언어 (전 화면 단일 팔레트)

**대원칙: 색상(hue)은 속성 전용 예약 채널이다.** 위협·상태·픽업·배경은 hue가 아니라 **명도·모양·레이어·렌더모드**로 구분한다.

#### 1.1 4속성 — 색 + 글리프 (색은 보조, **글리프가 진실**)

| 속성 | 키 | 색 | L* | **글리프 = 탄 실루엣** | 근거 |
|---|---|---|---|---|---|
| **노말** | `Q` | `#DCE6F0` | 91 | **● 원** | 중립·기본 |
| **불** | `W` | `#FF6B2C` | 62 | **▲ 삼각(위)** | 불꽃 |
| **물** | `E` | `#31C0FF` | 73 | **◆ 마름모** | 물방울 |
| **풀** | `R` | `#5FD13B` | 74 | **✚ 십자(사엽)** | 잎 |

- **6px에서도 100% 구분되는 4실루엣**(원/삼각/마름모/십자)만 채택. 육각형·별 등 "원과 헷갈리는 형태"는 금지.
- **한 hue = 한 속성, 전 표면 공통**: 플레이어 탄 · 적 외곽선 · 적 글리프 · 히트박스 도트 · 스탠스 링 · 슬롯 pip · 속성 레벨 HUD · 드래프트 카드 · 데미지 숫자 · 상성 처치 FX · 테마 고지 · 보스 부위 외곽선.
- **키 순서 = HUD 순서 = 카드 순서 = 상성표 순서 = `Q W E R` (노말·불·물·풀)** 로 **전 문서·전 UI 통일**. (기존 본문의 "물·불·풀" 나열 순서는 이 순서로 정정한다 — 표면마다 순서가 다르면 머슬메모리가 성립하지 않음.)
- 불(L*62) ↔ 풀(L*74)은 적록색약에서 색상만으로는 취약 → **글리프가 보증**하고, 색맹 모드(§11)가 명도 사다리로 완전 분리.

#### 1.2 예약 채널 (속성이 아닌 것들)

| 채널 | 색 | 어디에만 | 절대 금지 |
|---|---|---|---|
| **적 탄 · 텔레그래프** | 자홍 `#FF2E88` | 위협 전용 | 4속성 색 사용 금지 |
| **하드 외곽선** | `#0A0C10` 2px | 적 탄·텔레그래프 전용 | 플레이어 요소에 사용 금지 |
| **내 상태 이상 · 저체력** | 호박 `#FFB000` | **플레이필드 밖**(화면 가장자리 밴드) + 온-플레이어 아이콘 | 탄·적·배경 사용 금지 |
| **픽업** | 코인 금 `#FFC93C` / XP 백 `#EAF2FF` | 납작·무광·하단 레이어 | 글로우·외곽선 금지 |

```json
"palette": {
  "element": { "normal": "#DCE6F0", "fire": "#FF6B2C", "water": "#31C0FF", "grass": "#5FD13B" },
  "threat":  { "enemyBullet": "#FF2E88", "bulletCore": "#FFFFFF", "outline": "#0A0C10", "telegraph": "#FF2E88" },
  "status":  { "band": "#FFB000" },
  "pickup":  { "coin": "#FFC93C", "xp": "#EAF2FF" },
  "bg":      { "maxSaturation": 0.25, "maxLightness": 0.28 }
}
```

### 2. 적 탄 언어 (예약 팔레트 + 예약 렌더모드)

**적 탄에는 속성이 없다(확정) → 적 탄은 단 하나의 정체성을 독점한다.**

| 속성 | 플레이어 탄 | **적 탄** |
|---|---|---|
| 색 | 4속성 hue | **자홍 고정** |
| 코어 | 같은 hue의 밝은 코어(L*+25) | **흰 스페큘러 점** |
| 엣지 | 소프트(글로우) | **하드 + 검은 외곽선 2px** |
| 합성 | additive | **불투명(source-over)** |
| 알파 | ≤ 0.80 | **1.0 고정** |
| 레이어 | 4 | **9 (최상단)** |
| 모양 | 속성 글리프(●▲◆✚) | **원 / 육각(상태이상)** |

> **3초 안에 배우는 한 문장: "부드럽게 빛나면 내 것, 테두리 있고 딱딱하면 적 것."**

#### 2.1 거동별 적 탄 실루엣 (자홍 계열 내부에서 모양으로 구분)

| 거동 | 실루엣 | 부가 |
|---|---|---|
| 직선탄 | 작은 원 r=4 | — |
| 확산탄 | 작은 원 r=4 | — |
| **유도탄** | **마름모 + 3장 잔상 꼬리** | 꼬리로 "쫓아온다"가 즉시 읽힘 |
| **레이저** | 라인(발사 시 흰 코어) | §3 |
| 설치 장판 | 링(속 빈 원) | 지면 레이어 1 |
| **둔화 탄** | **육각형, 크기 ×1.5, 저속 회전, 흰 점선 궤적** | 속도 ×0.6 |
| **스턴 탄** | **육각형 + 흰 이중 링 4Hz 점멸, 크기 ×1.8** | 속도 ×0.6, 리드타임 최대 |

#### 2.2 가독성 불변식 (시뮬 공정성 검사 항목)

```json
"legibilityInvariants": {
  "enemyBulletMaxSpeedPxGameSec": 420,
  "enemyBulletMinSpawnRadiusPx": 140,
  "statusBulletSpeedMul": 0.6,
  "minGapWidthPx": 56,
  "enemyBulletMinRadiusPx": 4,
  "enemyBulletOutlinePx": 2
}
```
- **점블랭크 스폰 금지**: 적 탄은 플레이어 반경 `minSpawnRadiusPx` 이내에서 생성 불가.
- **통과 틈 하한**: 모든 탄막 패턴은 폭 `minGapWidthPx` 이상의 통과 가능 틈을 항상 최소 1개 유지(AI 생성 패턴의 **거부 조건**, 헤드리스 시뮬이 자동 검증).
- 이 3개가 "관대함"의 **기계 검증 가능한 정의**다. 정성 서술을 대체한다.

### 3. 텔레그래프 규칙

#### 3.1 형태 언어 — 표시 상태가 곧 의미

| 상태 | 표현 | 의미 |
|---|---|---|
| **곧 아플 곳** | **자홍 점선 외곽선 + 내부 알파 0.5, 채움(fill) 금지** | 텔레그래프 |
| **지금 아픈 것** | **자홍 채움 + 불투명 + 검은 외곽선** | 실탄 |
| 전환 순간 | **흰색 1프레임 플래시** | 발사 |

> 점선 = 미래, 실선 채움 = 현재. 이 2상태 규칙 하나로 6종 거동 전부를 커버한다.

#### 3.2 거동별 텔레그래프 (모양 어휘 확정 — 애드혹 금지)

| 거동 | 텔레그래프 모양 | `leadSecGame` |
|---|---|---|
| 직선탄 | 발사구 자홍 원 확대 | **0.40** |
| 확산탄 | **점선 부채꼴**(실제 각도 그대로) | **0.60** |
| 유도탄 | **플레이어 위 조준 브래킷 4각 수축** | **0.55** |
| **레이저** | **점선 라인 → 폭 0→최종폭 보간** | **1.20** |
| 설치 장판 | **지면 점선 원 수축**(레이어 1) | **0.90** |
| 둔화 탄 | 발사구 원 + **호박 테두리 링** | **0.80** |
| **스턴 탄** | 발사구 + **화면 가장자리 호박 1회 사전 펄스** | **1.50** |
| 중간보스 패턴 | 전신 자홍 림 라이트 | **1.20** |
| **보스 대형 패턴** | **전신 광휘 + 패턴 형상 점선 프리뷰** | **1.50** |

```json
"telegraph": {
  "leadSecGame": { "straight":0.40, "spread":0.60, "homing":0.55, "laser":1.20,
                   "groundZone":0.90, "statusSlow":0.80, "statusStun":1.50,
                   "midbossPattern":1.20, "bossPattern":1.50 },
  "minLeadSecGame": 0.40,
  "alpha": 0.50, "strokePx": 2, "dashPx": [6,5], "fireFlashFrames": 1,
  "emphasisBySpeed": { "1.0":1.0, "1.5":1.15, "2.0":1.30, "3.0":1.50 }
}
```
- `minLeadSecGame: 0.40` = **모든 공격의 절대 하한**. AI 생성 패턴이 이보다 짧은 값을 내면 **로드 시 거부**.
- 각 거동은 정확히 하나의 텔레그래프 모양만 갖는다 → 스테이지·보스가 늘어도 학습이 이월된다.

#### 3.3 ★ 배속과 가독성 — 명시적 결정 ★

문서가 지금까지 무의식적으로 미뤄온 결정을 여기서 확정한다.

- **회피 예산(리드타임·탄속·스폰 거리)은 전부 게임시간으로 정의하고 배속에 그대로 따른다. 실시간 하한 없음. 예외 없음.**
  → "난이도 = 게임 클럭 하나" 원칙을 훼손하지 않고, 배속별 게임플레이 분기를 만들지 않으며, 시뮬이 게임시간 단일 기준으로 남는다.
- **무-트위치 기둥은 `Normal`에서 보장된다**(문서 L119 "난이도는 옵트인"의 정식화). Hell/Disaster에서 어려워지는 것은 **읽고 반응할 시간**이지 **무엇인지 알아보는 것이 아니다.**
- **가독성 채널(색·모양·레이어·알파·오디오)은 배속과 무관하게 항상 동일.**
- 배속 보정은 **렌더 전용**으로만 허용(게임플레이 무영향):
  - `telegraph.emphasisBySpeed` — 배속이 높을수록 텔레그래프 **선 굵기·알파만** 증가.
  - `render.fxMinRealMs: 50` — 순수 장식 플래시가 고배속에서 소실되지 않도록 **표시 시간에만** 실시간 하한(판정 무관).

### 4. 배경 · 아레나 대비 규칙 (화산=빨강 문제 원천 차단)

| 규칙 | 값 |
|---|---|
| 배경 채도 상한 | `bg.maxSaturation: 0.25` |
| 배경 명도 상한 | `bg.maxLightness: 0.28` |
| 배경 hue | 4속성 hue **사용 가능하나 위 상한 내에서만** (화산 = 붉은 차콜, 늪·숲 = 녹회색 차콜) |
| 자홍 hue | **배경 사용 절대 금지** |
| 패럴랙스 | `bg.parallaxLayers: 2`, `bg.maxScrollPxGameSec: 90` — **적 탄 최저속(`≥140`)의 절반 이하**로 강제 → 배경이 탄처럼 안 보임 |
| 배경 대비 | 모든 배경 요소는 **엣지 대비 금지**(소프트만) → 하드 엣지 = 게임플레이 오브젝트 전용 |

> **결정적 규칙: 적 기체 본체는 항상 중립 차콜 `#20242C`.** 속성은 본체 색이 아니라 **외곽선 + 코어 글리프 + 림 라이트**로만 표현한다(§5). → 배경(저채도·저명도) ↔ 적 실루엣(명도 분리) ↔ 속성(고채도 라인 분리)의 3층 분리가 성립하고, "붉은 배경 위 붉은 불 적" 붕괴가 구조적으로 불가능해진다.

### 5. 적 속성 판독 (군중 속 <200ms)

| 채널 | 규칙 |
|---|---|
| **외곽선** | 속성색 2px (`a11y.outlinePxCvd`에서 3px) — 본체 크기 무관 항상 |
| **코어 글리프** | 본체 중앙 속성 글리프. 크기 = `max(6, bodyPx * glyph.bodyRatio)` |
| **림 라이트** | 진행 방향 반대쪽 속성색 소프트 림(알파 0.35) |
| **본체** | 중립 차콜 `#20242C` — **속성별로 본체를 칠하지 않는다** |

```json
"glyph": { "bodyRatio": 0.42, "minPx": 6, "maxPx": 22, "cvdScale": 1.5 }
```
- **겹침(새떼 30+)**: 외곽선이 각 개체를 분리하고, 글리프는 최상위 개체만 렌더(`glyph.occludedSkip: true`) → 겹친 덩어리가 단색 뭉텅이가 되지 않는다.
- **혼합 30% 적**: 테마와 다른 속성은 **외곽선이 테마색과 다른 hue** + 글리프가 다른 실루엣 → 같은 색 군중 속에서 "다른 모양 테두리"로 튄다. 추가 표식 불필요.
- **엘리트**: 본체 크기 무관 **회전하는 이중 외곽선(속성색)** + 개체 위 **속성색 HP 바** + 상시 글리프. 중간보스와의 구분 = 엘리트는 HP 바가 개체에 붙고, 중간보스는 화면 상단 바.

#### 5.1 복합 보스 부위 판독

- **부위마다**: 속성색 외곽선 3px + 부위 중심 글리프 + 부위 위 소형 HP 바(속성색).
- **피격 귀속**: 내 탄이 맞은 부위의 외곽선이 **0.1초 밝게 점멸** → "지금 여기를 때리는 중"이 항상 명시.
- **몸체=노말**: 은색 외곽선 + ● 글리프 → "스탠스 무관"이 색으로 자명.
- **파괴된 부위**: 외곽선 소멸 + 본체 `#3A3A3A` + 연기 파티클 + 글리프 제거 → 살아있는 부위와 절대 안 헷갈림.

### 6. 히트 피드백 — ×2 / ×1 / ×0.5 는 **다른 감각**이어야 한다

#### 6.1 3배율 표현표

| | **×2 (상성)** | **×1 (중립)** | **×0.5 (저항)** |
|---|---|---|---|
| **적 반응** | 속성색 화이트-핫 1프레임 + **0.04초 임팩트 프리즈(스케일 1.12)** | 짧은 백색 플래시 | **회색 방패 호(arc) 1회** |
| **파티클** | 속성색 3개, 큰 것 | 1개 | **0개** |
| **처치 FX** | **속성색 대형 파열 + 흰 링 확장** | 소형 파열 | (해당 없음) |
| **모양 마커** | — (파열 자체가 마커) | — | **회색 방패 호** |
| **숫자**(보스/중간보스/엘리트 한정) | 속성색, ×1.5 크기, `×2` 접미 | 은색, ×1.0 | 회색, ×0.7, `×½` 접미 |
| **SFX** | 저역 **"크런치"**(짧고 두꺼움) | 얇은 "틱" | **"틴"**(금속 튕김, 명확히 다른 음색) |

> **핵심**: ×0.5는 "약한 히트"가 아니라 **"튕겼다"**로 표현한다. 파티클 0 + 방패 호 + 금속 튕김음 → 플레이어가 무기를 탓하지 않고 **스탠스를 탓한다**. 정보 루프가 여기서 닫힌다.
> **혼재 히트**(1번 무기 ×2, 3번 무기 ×1이 같은 프레임): 배율 마커는 **그 프레임의 최고 배율 1개만** 표시(`markerPolicy: "maxOnly"`) → 같은 적에서 상충 신호가 뜨지 않는다.

#### 6.2 데미지 숫자 — **잡몹에는 없다 (확정)**

밀도 붕괴의 1순위 원인. 4무기 × 30적 = 초당 수백 개 = 적 탄 은폐 = I-4 위반.

```json
"hitFx": {
  "numberTargets": ["boss","midboss","elite"],
  "numberAggregateSec": 0.15,
  "numberMinPx": 14,
  "numberOutlinePx": 2,
  "numberLifeSec": 0.7,
  "numberDriftPx": 18,
  "markerPolicy": "maxOnly",
  "markerCooldownSecPerEntity": 0.25,
  "superFreezeSec": 0.04,
  "superFreezeScale": 1.12,
  "particles": { "super": 3, "neutral": 1, "resist": 0 }
}
```
- 숫자는 **0.15초 윈도우로 개체별 합산 후 1개만 팝** → 보스전에서도 숫자가 4개를 넘지 않는다.
- 숫자는 **개체 실루엣 밖(위쪽)에만** 배치, `caps.damageNumbers: 10` 초과 시 오래된 것부터 제거.
- 임팩트 프리즈는 **개체 단위 렌더 연출**이며 게임 클럭을 멈추지 않는다(보스 타이머·시뮬 결정성 무영향).

### 7. 스탠스 · 부여 판독 ★게임에서 가장 새로운 것★

"투자 N만큼 슬롯 순서대로 앞의 N개"는 인지적으로 추상적이다 → **눈이 있는 곳(기체)에 답이 있어야 한다.**

#### 7.1 기체에 붙은 3중 표시 (HUD 구석은 플레이 중 사실상 안 보임)

| 요소 | 규격 | 의미 |
|---|---|---|
| **히트박스 도트** ★ | **흰 코어 r=3 + 현재 스탠스 색 링 1px**, 레이어 10(적 탄보다 위) | 항상 표시(Touhou식). **눈이 항상 보는 유일한 점이 곧 스탠스 표시기.** 투자 0인 스탠스면 **은색**(= 실제로 아무것도 부여 안 됨 → I-2 준수) |
| **기체 림/후미 오라** | 현재 스탠스 색 소프트 글로우, 알파 0.4 | 원거리 확인용 |
| **슬롯 스트립** ★ | 기체 하단 `+14px`, pip 7px × 4, 간격 2px (총 34px) | **부여된 앞 N개 = 속성색 채움 + 글리프 / 나머지 = 은색 ● (노말)**. **슬롯 순서 = 좌→우 = 부여 우선순위**가 이 스트립으로 상시 가시화 |

> 슬롯 스트립은 HUD에도 큰 버전으로 복제하되, **진실의 원천은 기체 옆**이다. (배치는 [화면·아레나/HUD] 섹션 소관, 존재·내용은 여기서 확정.)

#### 7.2 전환 순간 피드백 (쿨다운 0 = 신뢰가 즉시 필요)

| 케이스 | 연출 |
|---|---|
| **투자 있는 속성으로 전환** | 기체 중심 **속성색 링 r=0→28px, 0.18초 확장 후 소멸** + 림/도트/스트립 **즉시 갱신** + SFX(속성별 음정) |
| **투자 0인 속성으로 전환** ★ | 링이 속성색으로 시작 → **확장 중 회색으로 탈색**되며 소멸 + 스트립 전부 은색 유지 + **"빈" 저음** | 
| **노말(Q) 스탠스** | **은색 링 정상 확장**(탈색 없음) + 스트립 전부 은색 | 

> 투자 0 전환의 **탈색 링**이 "이 속성엔 투자가 없다"를 첫 판에 말없이 가르친다. 동시에 `Q`(정상 은색 링)와 `투자 0인 W/E/R`(탈색 링)이 **시각적으로 구분**되어, Q키가 시각적 no-op가 아니게 된다.

```json
"stance": {
  "ringExpandSec": 0.18, "ringMaxRadiusPx": 28, "ringStrokePx": 2,
  "emptyDesatSec": 0.18,
  "dotRadiusPx": 3, "dotRingPx": 1,
  "auraAlpha": 0.40,
  "pipPx": 7, "pipGapPx": 2, "pipOffsetYPx": 14
}
```

#### 7.3 비행 중 탄 — I-2의 직접 귀결
**화면에 그려진 색은 그 개체가 실제 적용할 배율과 항상 일치해야 한다.** 스탠스 전환 시 이미 나간 탄/기뢰/오라의 **귀속 규칙은 [속성 시스템] 섹션이 결정**하되, 어떤 규칙을 택하든 **렌더는 그 규칙을 즉시·정확히 반영**한다. "색은 그대로인데 배율만 바뀜"(또는 그 반대)은 구현 버그로 취급한다.

### 8. 상태이상 · 피격 · 저체력 — 호박 채널 3중 표시

**호박 `#FFB000` = "내 상태가 나쁘다"** 단일 채널. 플레이필드 안에는 절대 안 나오고(가장자리 밴드 + 온-플레이어 아이콘만), 코인 금색과는 **영역이 분리**되어 충돌하지 않는다.

| 층 | 둔화 | 스턴 | 저체력 |
|---|---|---|---|
| **① 사전 (적 tell)** | 육각탄 ×1.5 + 회전 + 흰 점선 궤적, 리드 0.80s | 육각탄 ×1.8 + 흰 이중 링 4Hz, **리드 1.50s + 화면 가장자리 사전 펄스 1회** | — |
| **② 온-플레이어** | 기체 위 12px **∿ 아이콘 + 잔여 감소 바** | 기체 위 **✳ 아이콘 + 잔여 바** + 기체 주위 흰 링 4Hz | — |
| **③ 화면 가장자리** | 호박 밴드 12px, **1Hz 맥동** | 호박 밴드, **4Hz 점멸** | 호박 밴드 **1Hz 호흡**(≤30%) → **2Hz + 두께 ×1.5**(≤15%) |
| **④ 전용** | ★ **기체 뒤 잔상 3장(알파 0.30)** = "느리다"가 시각적으로 명시 → **벽에 박은 것과 절대 안 헷갈림** | 입력 무시 중임을 링 점멸이 명시 → **"입력 씹힘"으로 안 읽힘** | 저음 심박 SFX |

- **저항/해제 유틸 보유·발동**: 온-플레이어 아이콘 옆 **작은 호박 방패 pip 상시 표시**(보유) → 발동 시 **아이콘이 산산조각 + "챙"** (구매한 것이 일한다는 것이 보임).
- **피격 vs 실드 흡수는 반드시 다르게** (점수 규칙이 여기에 물림):

| | 표현 |
|---|---|
| **실제 피격** | 화면 **흰 플래시 1프레임(알파 ≤0.35)** + 화면 흔들림 6px + 기체 8Hz 점멸(무적시간 동안, 알파 0.35↔1.0) + 저음 임팩트 |
| **실드 흡수** | **흰 육각 링 파열 + 링 소멸 + "챙"**, 플래시·흔들림 **없음** |
| **실드 보유 중** | 기체 주위 **흰 얇은 육각 링 상시** |

> 무적시간의 존재·길이는 [플레이어] 섹션 소관. **시각 규칙(8Hz 점멸 + 흰 링)은 여기서 확정**하며, 무적시간이 존재한다면 반드시 이 표현을 갖는다(보이지 않는 무적시간 금지).

```json
"status": {
  "iconPx": 12, "iconOffsetYPx": -16, "barPx": [12,2],
  "edgeBandPx": 12, "edgeBandAlpha": 0.55,
  "slowPulseHz": 1.0, "stunPulseHz": 4.0,
  "slowGhostCount": 3, "slowGhostAlpha": 0.30,
  "iFrameBlinkHz": 8.0,
  "lowHp": { "threshold": 0.30, "criticalThreshold": 0.15, "pulseHz": 1.0, "criticalPulseHz": 2.0, "criticalBandMul": 1.5 }
}
```

#### 9.2 ★ 플레이어 FX 하드캡 (핵심 구현 규칙) ★

**플레이어의 모든 발광 FX(레이어 3~4)는 별도 오프스크린 캔버스에 그린 뒤, `globalAlpha = render.playerFxCompositeAlpha`로 화면에 1회 합성한다.**
→ 무기가 몇 개든, 진화체가 얼마나 화려하든, **플레이어 쪽 밝기 총량이 수학적으로 상한**을 넘지 못한다. 스테이지 5~6 맥스 빌드에서 화면이 하얗게 날아가는 실패 모드가 **구조적으로 발생 불가**. 전체 화면 `drawImage` 1회 = 바닐라 60fps에서 무시 가능한 비용.

#### 9.3 예산 · 초과 시 동작 (시뮬 결정성 필수)

```json
"render": {
  "refPlayfieldPx": [640, 960],
  "playerFxCompositeAlpha": 0.45,
  "playerBulletMaxAlpha": 0.80,
  "playerBulletMaxRadiusPx": 10,
  "particleMaxAlpha": 0.60,
  "particleMaxLifeSec": 0.35,
  "fxMinRealMs": 50,
  "targetFps": 60
},
"caps": {
  "enemyBullets": 320, "playerBullets": 220, "enemies": 90,
  "pickups": 200, "particles": 400,
  "telegraphsVisible": 6, "damageNumbers": 10, "effectMarkers": 12,
  "overflow": {
    "enemyBullet": "rejectSpawn",
    "playerBullet": "rejectSpawn",
    "telegraph": "deferAttack",
    "particle": "evictOldest",
    "damageNumber": "evictOldest",
    "effectMarker": "evictOldest"
  }
}
```
- **게임플레이 개체(적 탄·플레이어 탄)** 초과 = `rejectSpawn`. **오래된 것 재활용 금지** — 날아가던 탄이 사라지면 화면이 거짓말한다(I-2 위반).
- **텔레그래프** 초과 = `deferAttack` — **공격 자체를 연기**(I-3). 위기 세션(새떼)에서 화면이 텔레그래프로 뒤덮이는 것도 이 예산이 막는다.
- **순수 FX**(파티클·숫자·마커) 초과 = `evictOldest`. 시뮬 무관.
- 밀도 예산은 **논리 플레이필드 640×960 기준**으로 저작. 아레나 규격은 [화면·아레나] 섹션 소관이며, 변경 시 **면적 비례 스케일**(`caps.areaScale` 1개 키로 일괄).

### 10. 픽업 판독 (유도탄과 절대 안 헷갈리게)

**충돌 위험**: 자동수집으로 플레이어에게 다가오는 픽업 ↔ 유도탄. 둘 다 "나를 향해 온다".

| | 픽업 | 유도탄 |
|---|---|---|
| 색 | 금 `#FFC93C` / 백 `#EAF2FF` | 자홍 |
| 외곽선 | **없음** | 검은 2px |
| 글로우 | **없음(무광·납작)** | 있음 + 잔상 꼬리 |
| 레이어 | 2 (바닥 쪽) | 9 (최상단) |
| 크기 | ≤6px | ≥8px |
| **운동** ★ | **자석 반경 밖: 플레이어를 절대 안 쫓음**(스크롤 따라 하강만). **반경 안: 직선 스트릭으로 0.2초 내 도달** | **느리게 곡선으로 추적** |

> **한 문장: "천천히 곡선으로 쫓아오면 100% 탄. 순식간에 직선으로 빨려오면 100% 보상."** 운동 특성이 겹치는 구간이 존재하지 않는다.

- **자석 반경 가시화**(상점에서 사는 것이 보여야 함): 기체 주위 **은색 점선 원, 알파 0.12 상시** + 반경 증가 구매 시 **0.5초 확장 연출**.
- 모양: 코인 = 원반(중앙 구멍), XP = 작은 마름모. 회전으로 "아이템"임을 명시.

```json
"pickup": { "maxPx": 6, "snapSec": 0.20, "magnetRingAlpha": 0.12, "magnetRingDashPx": [3,4], "glow": false, "outline": false }
```

### 11. 색맹 · 접근성

**목표 선언**: 1차 타깃 = **적록색약(deuteranopia/protanopia, 남성 ~8%)**. 2차 = 청황(tritan) 및 완전 색각이상 — **글리프 채널이 이를 100% 커버**한다.

> **철학: 색맹 모드의 목표는 "색으로 4속성을 구분시키는 것"이 아니다. 4개 카테고리를 어두운 배경 위에서 색만으로 이색각자에게 분리시키는 것은 이론적으로 불가능하다. 목표는 "색이 완전히 실패해도 게임이 100% 성립하는 것"이며, 그 보증이 §1.1의 글리프다.** 색맹 모드는 그 보증을 강제로 강화하는 스위치일 뿐이다.

`a11y.cvdMode: "off" | "cvd" | "mono"`

| | `off` | `cvd` | `mono` |
|---|---|---|---|
| 팔레트 | 기본 | **명도 사다리 팔레트**(아래) | 명도 사다리 + 채도 0 |
| 글리프 크기 | ×1.0 | **×1.5** | ×1.5 |
| 글리프 렌더 | LOD 적용 | **항상 렌더**(LOD 무시) | 항상 |
| 적 외곽선 | 2px | **3px** | 3px |
| **플레이어 탄** | 속성색 + 글리프 실루엣 | **글리프 실루엣이 주(主), 색은 보조** | 글리프만 |
| 배경 명도 상한 | 0.28 | **0.22** | 0.22 |
| HUD/카드/고지 | 색 + 글리프 | 색 + 글리프 + **텍스트 라벨 강제** | 글리프 + 라벨 |

**명도 사다리 팔레트** (`mono`에서도 4단 분리):
```json
"paletteCvd": {
  "normal": "#F2F6FA",  // L* 96
  "fire":   "#FFA83A",  // L* 76
  "water":  "#2E86E0",  // L* 55
  "grass":  "#1F6B3A",  // L* 40
  "bgMaxLightness": 0.22
}
```
- 노말 96 / 불 76 / 물 55 / 풀 40 → **인접 간격 ≥15 L\*** → 완전 흑백에서도 4단 구분.
- 각 속성 탄은 **같은 hue의 밝은 코어(L\*+25)** 를 가지므로 어두운 풀(L*40)도 배경(≤22) 위에서 확실히 보인다.

**광과민 · 모션**
```json
"a11y": {
  "cvdMode": "off",
  "reduceFlash": false,
  "fullscreenFlashMaxPerSec": 3,
  "fullscreenFlashMaxAlpha": 0.35,
  "screenShake": true,
  "shakeMaxPx": 6

```
- 풀스크린 플래시는 **초당 3회 · 알파 0.35 하드캡**(폭탄·노바·피격 전부 이 예산 공유).
- `reduceFlash: true` → 풀스크린 플래시 **전면 금지**, 화면 흔들림 0. 대체 표현: **화면 가장자리 밴드 1회 펄스**.
- **폭탄**: 흰 플래시가 아니라 **자홍→흰 링 1회 확장**으로 "탄 소거"를 표현 → 소거되는 탄을 플래시가 가리는 자기모순 회피.

**텍스트 렌더** (오프라인·저용량 → 웹폰트 금지)

### 12. 오디오 큐 (가독성 한정)

> 시각 채널이 밀도·색맹에 막혔을 때 **유일하게 살아남는 채널**. 여기서는 **비주얼과 1:1로 짝지어지는 큐만** 확정한다. BGM·믹스·에셋 방침은 [오디오] 섹션 소관.
> **방식: WebAudio 절차적 합성(에셋 0KB, 완전 오프라인, 바닐라)** — 저용량 목표와 충돌 없음.

| 큐 | 음색 | 짝지어진 비주얼 |
|---|---|---|
| 텔레그래프 시작(레이저/장판/보스) | 상승 스윕 + 리드타임만큼 지속 | 점선 도형 |
| **상태이상 탄 발사** | **하강 2음(전용)** | 육각탄 |
| **스턴 탄 발사** | **경고 상승음(최대 강조)** | 가장자리 사전 펄스 |
| **×2 히트** | **저역 크런치(짧고 두꺼움)** | 화이트-핫 + 파열 |
| ×1 히트 | 얇은 틱 | 백색 플래시 |
| **×0.5 히트** | **금속 "틴"(튕김)** | 회색 방패 호 |
| **스탠스 전환** | 속성별 음정 (Q/W/E/R = 4음) | 링 확장 |
| **투자 0 전환** | **"빈" 저음** | 탈색 링 |
| 피격 | 저음 임팩트 | 흰 플래시 + 흔들림 |
| 실드 흡수 | **"챙"** | 육각 링 파열 |
| 저체력 | 저음 심박 (임계별 1Hz/2Hz) | 호박 밴드 |
| 보스 타이머 빨간불 | 1Hz 틱 | (타이머 표시는 HUD 섹션) |
| 레벨업 | 상승 아르페지오 | 드래프트 등장 |

- **음소거 상태에서도 게임이 100% 성립해야 한다**(오디오는 이중화 채널이지 유일 채널이 아니다). 위 표의 모든 큐는 시각 짝을 반드시 갖는다.
- `audio.cueRateLimitPerSec: 12` — 동일 큐 초당 상한(밀도 시 소리 뭉개짐 방지).
- **재생 속도는 배속과 무관**(피치 시프트 없음) — 가독성 채널은 배속 불변(§3.3).

### 13. 온보딩 관점의 최소 보증 (심사자 30초)

이 섹션의 규칙만으로 **설명 없이** 다음이 전달되어야 한다(구현 시 체크리스트):

1. 3초 — "테두리 있고 딱딱한 자홍 = 나를 아프게 하는 것" (I-1)
2. 5초 — 점선 = 곧 위험, 실선 = 지금 위험 (§3.1)
3. 10초 — 히트박스 도트 색 = 내 스탠스 (§7.1)
4. 15초 — 기체 아래 4칸 pip = 지금 어느 무기가 부여됐나 (§7.1)
5. 20초 — ×2 크런치 + 파열 vs ×0.5 방패 + "틴" = 스탠스를 바꿔야 하는 이유 (§6.1)
6. 25초 — 투자 0 전환의 탈색 링 = "이 속성은 투자가 필요하다" (§7.2)

> 튜토리얼 스크립트·스테이지 1 편성은 [온보딩] 섹션 소관. **여기서 보증하는 것은 "가르칠 수 있는 화면"이 존재한다는 것**이다.

### 14. 이 섹션이 닫은 결정 (미결 없음)

- 속성 색 4종 + 글리프 4종 확정 · 전 표면 단일 팔레트 · 키 순서 `Q W E R`로 전 문서 통일
- 적 탄 = 자홍 + 검은 외곽선 + 최상단 불투명 (4속성 색 금지) — 렌더모드가 색맹과 무관하게 분리 보증
- 텔레그래프 = 점선/실선 2상태 + 거동별 모양 어휘 8종 + 리드타임 표 + 하한 0.40s + **I-3(텔레그래프 예산 = 공격 예산)**
- **배속 결정 명문화**: 회피 예산은 게임시간 고정(예외 없음), 무-트위치는 Normal에서 보장, 가독성 채널은 배속 불변, 배속 보정은 렌더 전용
- 배경 = 저채도·저명도 + 적 본체 중립 차콜 → "화산=빨강" 충돌 구조적 불가
- 히트 피드백 3배율 차별화 + **잡몹 데미지 숫자 폐지**(보스/엘리트 한정 + 0.15s 집계)
- 스탠스 = 히트박스 도트 + 기체 오라 + 기체 하단 슬롯 스트립 3중 + **투자 0 탈색 링**(Q키 시각적 의미 확보)
- 상태이상 = 호박 단일 채널 3중 표시 + **둔화 잔상**(벽 충돌과 분리) + 피격/실드 흡수 분리
- 픽업 = 운동 특성으로 유도탄과 완전 분리 + 자석 반경 가시화
- 레이어 14단 + **플레이어 FX 오프스크린 합성 하드캡**(4칸 상한의 실제 집행 장치) + 예산·초과 동작(결정적)
- 색맹 3모드 + 명도 사다리 팔레트 + 광과민 캡 + 텍스트 렌더
- 오디오 = 가독성 큐 13종, 절차적 WebAudio, 음소거에도 100% 성립
