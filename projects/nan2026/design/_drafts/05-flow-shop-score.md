## 런 플로우 · 상점 · 점수 · 게임오버 · 온보딩

> 이 섹션은 **런의 시작~종료까지 모든 화면과 전이**를 확정한다. 원칙: **구조는 여기서 확정, 밸런싱은 `flow.json` / `shop.json` / `score.json`의 숫자만 만진다.**
> 표기 `키: 기본값` = 튜너블 JSON 키(밸런싱 대상). 그 외 서술 = **설계 확정**(변경 = 설계 변경).

---

### 1. 전역 상태 기계

```
TITLE ──Space──> DIFFICULTY ──Space──> RUN_START
                     ↑Esc                  │
                     └──────────Esc────────┘
RUN_START → [스테이지 루프 ×6] → RESULTS
                                    │
스테이지 루프:                       │
  THEME_BANNER → MOB_PHASE → CRISIS → BOSS_INTRO → BOSS
     ↑                │                              │
     │            (DRAFT 즉시)                   (DRAFT 큐 적립)
     │                                              │
     │                                        보스 격파 ↓
     └── SHOP ← THEME_BANNER(다음) ← HEAL ← DRAFT_QUEUE ← STAGE_CLEAR
                                                        (스테이지 6이면 → RESULTS)

어디서든: Esc → PAUSE (게임 클럭 정지)
HP 0 또는 타이머 만료 → DEATH ──컨티뉴──> 직전 상태 재개
                          └──포기──> RESULTS
RESULTS ──Space(같은 난이도 즉시 재시작)──> RUN_START
        └─Esc──> TITLE
```

**진입 상태 = TITLE. 스테이지 6 보스 격파 = 클리어(승리). 엔드리스 없음.** 새로고침/이탈 = 런 소멸(세이브 없음, 확정).

### 2. 타이틀 · 난이도 선택 · 런 시작

#### 2.1 TITLE
| 요소 | 내용 |
|---|---|
| 배경 | **어트랙트 모드** — 시뮬용 헤드리스 봇이 스테이지 1을 *실제로* 플레이(입력만 봇이 주입). 봇은 AI 파이프라인에 이미 존재 → 재사용 비용 0에 가까움. 심사자가 첫 3초에 게임을 본다. |
| 표시 | 타이틀, 난이도별 최고점수 4줄, `Space = START`, 조작 키캡 그림(텍스트 아님) |
| 하위 항목 | `기록 초기화`(작게, 확인 1회) |
| `attractIdleSeconds` | `20` — 무입력 시 어트랙트 시작 |

#### 2.2 DIFFICULTY — 4종 **전부 처음부터 해금** (해금 그라인드 금지 = 기둥)
각 항목에 **배속 / 보스 타이머 실제시간 / 예상 런 길이 / 점수 배율 / 해당 난이도 내 최고점수**를 동시에 표시.
→ "Hell = 적이 세다"가 아니라 **"Hell = 15분 스피드런"**임이 선택 시점에 읽혀야 함. 난이도는 런 시작 후 **변경 불가**.

#### 2.3 RUN_START — 시작 로드아웃 (확정)
| 항목 | 값 |
|---|---|
| 무기 | **고정 1개: 포워드/벌컨 Lv1 → 슬롯 1.** 슬롯 2~4 비어 있음. (t=0의 "선택"은 정보가 없어 가짜 선택 → 고정. 대신 온보딩이 스크립트 가능해짐) |
| 속성 투자 | 물/불/풀 전부 **0** |
| 스탠스 | **Q(노말)** |
| HP | `playerMaxHp` 만피 |
| 스톡 | 코인 `0` · 폭탄 `startBombStock: 1` · 리롤 `0` · 실드 `0` · 시간토큰 `0` |
| 레벨 | 1 |
| 시드 | 런 시작 시 1회 발급, 결과 화면에 표시(시뮬 재현용) |
| 스테이지 1 테마 | 테마 풀에서 랜덤 추첨. 단 **노말 테마 제외**(`stage1ExcludesNormalTheme: true`) — 온보딩이 상성을 가르쳐야 하므로 |

### 3. 스테이지 시퀀스

#### 3.1 THEME_BANNER — 정보 기반 선택의 유일한 전달 지점
- 전체화면 배너 `themeBannerSeconds: 2.5`, **Space로 스킵 가능**.
- 표시: 테마 이름 + **정확한 구성비 바**(예: `물 70% / 불 12% / 풀 12% / 노말 6%`) + 4속성 글리프.
  → **수치를 숨기지 않는다.** "대략만 보여주기"는 정보 기둥에 대한 배신.
- 배너에 **상성 미니휠**이 항상 함께 그려지고, 이 테마에 ×2인 속성 글리프가 하이라이트된다. (텍스트 0)
- 배너 종료 후에도 **HUD에 테마 뱃지 상시 잔류**(구성비 축약형).
- 위치: **부분 회복 뒤, 상점 앞** → 다음 스테이지를 알고 사는 상점.

#### 3.2 MOB_PHASE — `mobPhaseSeconds: 120` (게임시간 고정 타이머)
- 웨이브 스크립트에 따라 스폰. **스킵 없음.**
- **파밍의 리스크 = 시간이 아니라 위치다.** 적은 아래로 흘러 내려가 **화면 이탈 = 소멸 = 그 XP 영구 소실**. XP를 얻으려면 이탈 전에 죽여야 하고, 그러려면 **위로 밀고 올라가 스폰 라인에 접근**해야 한다 = 탄 밀도가 높은 곳. 이것이 리스크/리워드의 실제 메커니즘이며 "의미 있는 이동" 기둥과 일치.
- HUD에 페이즈 잔여 시간 표시(게임시간).
- **중간 보스**: `midBossCountPerStage: [1, 2]`, `midBossWindow: [30, 85]` 사이 진입. **`midBossLingerSeconds: 30` 후 스스로 이탈(퇴장)**. 진행을 막지 않음.
  - "선택적"의 정확한 의미 = **잡는 것이 선택**(회피는 선택이 아님 — 있는 동안엔 쏜다). 무시 = 큰 XP + 확정 코인을 포기.
  - 진입 시 3초 예고 배너 + SFX.

#### 3.3 CRISIS(새떼) — `crisisStartAtSeconds: 95`, `crisisDurationSeconds: 25`
- 잡몹 페이즈 **내부**의 마지막 25초. 매 스테이지 필수.
- `crisisWarnLeadSeconds: 3` 전 경고 배너 + 사운드 스팅 + 화면 가장자리 펄스 → **예고 없는 스파이크 금지**(관대 텔레그래프 기둥).
- 이 구간부터 **드래프트는 큐에 적립**(§4.1).

#### 3.4 페이즈 전환 (잡몹 → 보스)
- `t = mobPhaseSeconds`에 스폰 정지 → **잔여 잡몹은 보스 등장 연출 중 아래로 퇴장·소멸.**
- **미수집 드랍(코인·XP)은 전환 순간 전량 자동 회수**(`sweepPickupsOnPhaseEnd: true`) → 줍기 노가다 0, 전환 손실 0.

#### 3.5 BOSS_INTRO → BOSS
- `bossEntranceSeconds: 3` 연출. **연출 중 보스 무적 + 발사 없음 + 타이머 정지.**
- 연출 끝나는 프레임 = **타이머 시작 + 보스 피격 가능**.

### 4. 레벨업 드래프트

#### 4.1 정지 규칙 ★ (핵심 결정)
| 구간 | 드래프트 |
|---|---|
| 웨이브 구간 (`0:00 ~ crisisStartAtSeconds`) | **즉시 발생. 게임 클럭 완전 정지**(적·탄·스폰·페이즈 타이머 전부 정지). 무제한 사고 시간. |
| 위기 세션 · 보스 페이즈 | **발생하지 않음. 큐에 적립.** HUD에 `LV UP ×N 대기` 배지. |
| STAGE_CLEAR 직후 | **큐를 전부 순차 소화**(§5 순서 1번) |

**왜 이렇게:**
- 보스전 = **가져온 빌드의 DPS 시험지**. 시험 중에 빌드를 고치면 시험이 아니다.
- 타이머를 정지시키면 3분 허들이 무료로 무력화되고, 진행시키면 탄막 속에서 카드를 읽는 = **트위치** → 두 함정을 모두 회피.
- 위기 세션도 같은 이유(밀도 최고 구간에서 3초마다 정지 = 리듬 파괴).
- **부수 효과: 보스전 중 슬롯 재정렬이 원천 불가능해짐** → 부위마다 정지→재정렬로 "투자한 만큼만" 게이트를 우회하는 체스식 최적화가 구조적으로 봉쇄된다(§4.4).

동시 다중 레벨업 = **병합 없이 순차 드래프트**.

#### 4.2 화면 구성
- 카드 3장 가로 배치. **← →로 커서, Space로 확정.** 마우스 사용 안 함.
- 하단에 **무기 슬롯 4칸 상시 표시**(순서·현재 부여 상태 포함) → 슬롯 순서 규칙이 결정 시점에 보인다.
- 상단에 **테마 뱃지 + 현재 속성 투자(물/불/풀)** 표시.

#### 4.3 카드 정보 (필수 표기 — 정보 기둥의 집행 지점)
| 카드 종류 | 반드시 표기 |
|---|---|
| 새 무기 | 이름 · 거동 아이콘 · **거동 미니 프리뷰(루프 애니)** · **들어갈 슬롯 번호** · 현재 투자 기준 **부여 여부**(예: `→ 슬롯 3 · 불 스탠스 시 무속성`) |
| 무기 레벨업 | 대상 슬롯 · `Lv3 → Lv4` · **변화 델타 1줄** · Lv7 카드는 **`다음 = 진화` 마크** |
| 속성 레벨업 | 속성 글리프 · `불 +2 → +3` · **`불 스탠스: 2칸 → 3칸 부여` before/after 슬롯 그림** ← 이게 없으면 이 카드는 무의미한 숫자다 |
| 패시브 | 이름 · 효과 1줄 |
| 폴백(코인) | `+ 코인 ×N` — 유효 후보가 3장 미만일 때만 등장 (`draftFallbackCoins: 40`) |

#### 4.4 리롤 · 슬롯 재정렬
| 항목 | 규칙 |
|---|---|
| 리롤 | **상점에서 사는 소비형 스톡**(HUD 상시 표시). 드래프트 화면에서 **`R`** 키. **3장 전부** 다시 뽑음. **직전 3장은 후보에서 제외**(`rerollExcludesPrevious: true`) → 리롤은 도박이 아니라 수정. 스톡이 있는 한 한 드래프트에서 연속 사용 가능. `rerollMaxStock: 3` |
| 슬롯 재정렬 | **`Tab`으로 재정렬 모드 진입** → ←→ 선택, Space 집기/놓기, Tab/Esc 종료. **키보드 전용.** |
| 재정렬 가능 시점 | **레벨업 드래프트 화면에서만.** 일시정지 화면에서는 **불가.** → 보스전 중 재정렬 구조적 봉쇄. |
| 비용 | 없음 |

### 5. 스테이지 클리어 → 다음 스테이지 (전이 시퀀스, 순서 확정)

| # | 단계 | 규칙 |
|---|---|---|
| 1 | **STAGE_CLEAR 리절트** | 2초 자동, Space 스킵. 스테이지 소계 + **보스 잔여시간 vs 파(par) 막대** 1개(§9.3) |
| 2 | **DRAFT_QUEUE** | 큐에 쌓인 드래프트 전부 순차 소화. **여기가 슬롯 재정렬의 정규 창구.** |
| 3 | **HEAL — 부분 회복** | 자동. 최대 HP의 `stageClearHealPercent: 25`. 연출 1.5초, 스킵 가능 |
| 4 | **THEME_BANNER (다음 스테이지)** | §3.1 |
| 5 | **SHOP** | §6 |
| 6 | 다음 스테이지 시작 | |

**순서의 근거:** 빌드(2) → 실제 HP(3) → 다음 위협(4)을 **전부 알고 나서** 지갑을 연다(5). 회복이 상점 뒤면 물약 구매가 도박이 되고, 테마가 상점 뒤면 유틸 구매가 장님이 된다. 정보를 최대화한 뒤 결정하게 하는 것이 이 게임의 기둥이다.

상점 방문 = **스테이지 1~5 클리어 후 총 5회.** 스테이지 1 시작 전 상점 없음(코인 0). 스테이지 6 클리어 후 없음(런 종료).

### 6. 상점 (SHOP)

#### 6.1 진열 규칙 ★ — "전체 목록"도 "완전 랜덤"도 아니다
| 칸 | 내용 | 성질 |
|---|---|---|
| 1 | **리롤** | 소비형 스톡 |
| 2 | **회복 물약** | 즉시 사용 (구매 = 즉시 회복 `potionHealPercent: 30`) |
| 3 | **폭탄 스톡 +1** | 소비형 스톡 |
| 4 | **실드 +1** | 소비형 스톡 |
| 5–6 | **로테이션 유틸 ×2** | 아래 풀에서 추첨 |
| 7 | **나가기** | Space 확정 |

- **1~4번 = 생존 코어, 항상 같은 자리에 항상 존재** → 진열 운빨 없음, 탐색 없음, 머슬메모리.
- **5~6번 = 로테이션**(`rotatingSlots: 2`): 풀 = `방어력↑ / 최대체력↑ / 이동속도↑ / 코인·XP 자석 반경↑ / 상태이상 저항↑ / 시간 연장 토큰`. **직전 방문과 중복 금지.**
- → 전체 목록 상시 개방(= 6회 방문의 최적 구매 순서를 푸는 **바둑/체스**)도, 순수 랜덤(= 운빨)도 아닌 중간. 사용자가 명시적으로 싫어하는 두 실패 모드를 동시에 회피.
- **상점 리롤 없음**(랜덤 위에 랜덤 금지). **시간 제한 없음**(무-트위치). **컨티뉴는 상점에 없음** → 사망 시점에 제시(§8).

#### 6.2 표기 (기회비용의 가시화 ★)
각 칸에: 아이콘 · 이름 · 효과 1줄 · **보유/상한**(예: `2/3`) · **현재 가격** · **`= −N 점`** · **다음 가격**.
→ `남은 코인 = 점수`라는 이 게임의 중심 긴장은 **상점에서 점수 손실을 숫자로 보여줄 때만** 존재한다. 안 보여주면 그 긴장은 설계자 머릿속에만 있다.

#### 6.3 가격 · 상한
- 가격 상승 = **항목별 누적, 런 전체** (스테이지마다 리셋 안 함).
  `price = basePrice × priceGrowth ^ (그 항목의 런 내 구매 횟수)`, `priceGrowth: 1.6`
- 상한 (**무한 스택 금지** — 코인이 빌드가 되는 우회로 차단):

| 항목 | 키 | 기본 |
|---|---|---|
| 리롤 보유 | `rerollMaxStock` | 3 |
| 폭탄 보유 | `bombMaxStock` | 3 |
| 실드 보유 | `shieldMaxStack` | 2 |
| 시간 토큰 보유 | `timeTokenMaxStack` | 2 |
| 스탯 유틸(방어/최대HP/이속/자석/저항) 각각 | `utilMaxStacks` | 3 |
| 회복 물약 | 상한 없음 (가격이 곧 상한) | — |

- `상태이상 저항/해제`는 **하나의 항목: 저항(영구, 지속시간 감소 %)**으로 확정. "해제(소비형)"는 폐기 — 두 개를 한 칸에 적어두던 모호성 제거.
- 기본 가격 `basePrice.*` 전부 `shop.json`.

### 7. 보스 타이머 UX

| 항목 | 값 / 규칙 |
|---|---|
| 길이 | `bossTimerSeconds: 180` **게임시간** (전 난이도 동일 = 확정) |
| 표시 | **게임시간 그대로 표시.** ×3에서는 숫자가 실제보다 3배 빨리 떨어진다 — 이것이 스피드런 판타지 그 자체이고, 게임 내 모든 시간 단위와 일치한다. |
| 시작 | `bossEntranceSeconds` 연출 종료 프레임 |
| 정지 | PAUSE / DRAFT(보스 중엔 드래프트 없음 → 사실상 PAUSE만) |
| 경고 1 | `timerWarnSeconds: 60` — 노란색 + 완만한 펄스 |
| 경고 2 (**빨간불**) | `timerCriticalSeconds: 30` — 적색 + 화면 가장자리 비네트 + 심박 SFX. **"빨간불" = 이 상태를 가리키는 공식 정의.** |
| **보스 페이스 고스트** | 보스 HP 바에 **파(par) 고스트 마커** — "지금쯤 여기까지 깎았어야 함". `bossParGhost: true`. 텍스트 0, 구현 비용 거의 0, **20분 뒤에야 알게 되는 언더파를 지금 알려준다.** |
| 만료 | **확정 사망.** 1.5초 처형 연출 후 DEATH. 반격 불가 — 대신 60초/30초 2단 경고가 그 반격 창이었다. |

#### 시간 연장 토큰
| 규칙 | 값 |
|---|---|
| 발동 키 | **`Shift`** (왼손 새끼손가락, QWER에서 손 안 뗌) |
| 발동 조건 | **빨간불(잔여 ≤ `timerCriticalSeconds`)일 때만.** 그 외엔 입력 무시(오발 불가) → "빨간불"은 안내가 아니라 **제약**이다 |
| 효과 | `timeTokenSeconds: 30` **게임시간** 추가 (배속과 무관 — 게임 내 모든 시간과 일관) |
| 중첩 | 보유 `timeTokenMaxStack: 2`, 연속 사용 가능 |
| 대가 | **`timeTokenForfeitsTimeBonus: true`** — 이 보스의 **남은 시간 보너스가 0이 됨.** |
| 왜 | 대가가 없으면 코인으로 3분 허들을 삭제할 수 있다. 대가가 점수라서, 토큰은 **"이 판을 포기할래 점수를 포기할래"**라는 진짜 결정이 된다. |

### 8. 사망 · 컨티뉴

#### 8.1 DEATH 화면
트리거: **HP 0** 또는 **보스 타이머 만료** (잔기 개념 없음 — 확정).

표시:
1. **사인 명시** — `체력 소진` / `시간 초과` (아이콘 + 1단어). *왜 죽었는지 모르는 죽음을 만들지 않는다.*
2. 도달 스테이지 / 경과 시간 / 현재 점수 소계
3. **스테이지별 보스 잔여시간 vs 파 막대**(§9.3의 축소판) — 어디서 뒤처지기 시작했는지가 그림으로 보임
4. **컨티뉴 제시** (아래) / `포기 → 결과 화면`

#### 8.2 컨티뉴 — 상점 사전구매가 아니라 **사망 시점 제시**(오락실식)
| 규칙 | 값 |
|---|---|
| 횟수 | **런당 1회 하드캡** |
| 비용 | `continueCost: 150` 코인, **고정**(1회 캡이라 인상 불필요) |
| 부족 시 | 항목이 회색 + 가격 표시 → 다음 런의 학습(“아껴야 하는구나”) |
| **카운트다운** | **없음.** 무기한 대기. (오락실 감성은 카운트다운이 아니라 *동전을 낼 것인가*의 결정에서 온다. 카운트다운 = 트위치 = 기둥 위반) |
| 대상 | **두 사인 모두 컨티뉴 가능**(체력 소진 / 시간 초과) |

**부활 상태 (전부 확정):**
| 항목 | 값 |
|---|---|
| HP | 최대치 (`continueHealToFull: true`) |
| 화면 | **적 탄 전량 소거**, 잡몹은 유지 |
| 위치 | 아레나 하단 중앙, `continueIframeSeconds: 3` 무적(점멸) |
| 빌드 | 레벨·무기·속성·패시브·슬롯 순서 **전부 유지** |
| 코인 | `continueCost` 차감, 나머지 유지 |
| 스톡 | 폭탄·실드·리롤·토큰 유지 |
| **잡몹 페이즈 사망** | 그 자리에서 재개. 페이즈 타이머 유지 |
| **보스 페이즈 사망** | **보스전 재개. 보스 HP는 깎은 그대로 유지.** 타이머 = `max(현재 잔여, continueTimerRestoreSeconds: 60)` |
| **타이머 만료 사망** | 동일 — 보스 HP 유지 + 타이머 60초 부여 |

> 타이머를 3:00으로 리셋하지 않는 것이 핵심. 리셋하면 컨티뉴는 "6분 보스전"이 되어 게임의 유일한 허들이 코인으로 삭제된다. **60초 = 마무리할 수 있는 시간이지, 다시 싸울 수 있는 시간이 아니다.**

**상실:** `runPerfectBonus` + **모든 스테이지의 `stageNoHitBonus`**(런 전체 소급 무효). + 코인 150 소모 = 점수 −3000 자동. → 이중 비용.

### 9. 승리 · 결과 화면 · 점수

#### 9.1 점수 판정 규칙 ★ (숫자가 아니라 *규칙*)

| 항목 | **판정 규칙** | 키 | 기본 |
|---|---|---|---|
| 처치 점수 | 처치 즉시. 보스 **부위도 개체** → 자체 `scoreValue` | `enemy.scoreValue` | — |
| **상성 처치 보너스** | **막타 기준 아님.** 개체마다 `총 피해`·`×2 피해` 2개 누산기. 처치 시 `×2 피해 / 총 피해 ≥ superEffectiveDamageShare`면 성립 → 처치 점수의 `+superEffectiveKillMultiplier` | `superEffectiveDamageShare` / `superEffectiveKillMultiplier` | `0.5` / `0.5` |
| **남은 시간 보너스** | **게임시간 잔여 초 × 계수.** 게임시간 기준이므로 배속에서 이중 보상 없음. **시간 토큰 사용 시 그 보스는 0** | `timeBonusPerGameSecond` | `50` |
| 보스 격파 | 스테이지 보스 / 중간 보스 | `bossClearBonus` / `midBossClearBonus` | `2000` / `500` |
| **무피격 보너스** | **스테이지 단위** — 그 스테이지 전체(잡몹+위기+보스)를 피격 0으로 클리어 | `stageNoHitBonus` | `3000` |
| **퍼펙트 보너스** | **런 단위 = 6/6 스테이지 무피격.** (기존 문서에서 페널티 대상으로만 언급되던 유령 용어를 여기서 정의) | `runPerfectBonus` | `30000` |
| 클리어 보너스 | 스테이지 6 보스 격파 | `runClearBonus` | `10000` |
| 남은 코인 환산 | 런 종료 시 잔액 × 계수 | `coinToScore` | `20` |
| **× 난이도 배율** | 위 전부의 합계에 최종 곱 | `difficulty[].scoreMultiplier` | `1 / 1.6 / 2.5 / 4` |

**부속 판정 (전부 확정):**
- **실드로 막은 피격 = 피격 아님** (`shieldPreservesNoHit: true`). 실드의 문구는 "1회 피격 무효"이므로 무피격도 유지되어야 문구와 동작이 일치한다. 코인으로 무피격을 살 수 있지만 **코인 = 점수**라서 자기상쇄된다.
- **사망해도 점수는 집계된다** — 그때까지의 획득 + 남은 코인 환산 + 난이도 배율. (오락실 규칙)
- **왜 막타가 아닌 지분인가:** 막타 기준이면 최적 플레이가 *"막타만 ×2로 맞추기"*가 되어, 사용자가 명시적으로 싫어하는 **체스식 최적화 + 트위치 정밀도**를 동시에 소환한다. 지분 기준이면 최적 플레이는 그냥 **"제대로 된 스탠스로 싸우기"** = 설계 의도 그 자체. 누산기 2개 = 구현 비용 0.

#### 9.2 RESULTS 화면 (승리·사망 공용, 헤더만 다름)
- 헤더: `CLEAR` / `GAME OVER — 체력 소진` / `GAME OVER — 시간 초과`
- **6줄 항목별 집계 + 오락실식 순차 타이핑 애니메이션**(Space로 즉시 완료)
- **놓친 보너스를 회색으로 함께 표시** (예: `무피격 3/6 · 퍼펙트 ✕ · 시간토큰 사용 1회 → 시간 보너스 −1`)
  → 이 게임엔 언락이 없다. **결과 화면이 유일한 교사다.** "다음엔 어느 레버를 당길까"가 여기서만 학습된다.
- **스테이지별 보스 잔여시간 vs 파 막대 6개**(§9.3)
- 최종 점수 → **난이도별 최고점수 비교**. 신기록이면 전용 연출 + 사운드.
- 런 시드 표시(작게).
- 하단: `Space = 같은 난이도로 즉시 재시작` / `Esc = 타이틀`

#### 9.3 파(par) 막대 — 무-텍스트 페이스 진단
스테이지별로 `보스 잔여 시간`을 막대로, `stagePar[i]`(시뮬이 산출한 표준 잔여시간)를 고스트 라인으로. 4스테이지에서 처음 파선을 밑돌았다면 **"20분 전에 파밍이 모자랐다"가 그림 한 장으로 보인다.** 텍스트 0, 판정 0.

#### 9.4 재시작 루프
`Space` = **타이틀·난이도 화면을 경유하지 않고 같은 난이도로 즉시 새 런.** 죽자마자 다시 하고 싶은 충동과 손가락 사이에 메뉴를 놓지 않는다(오락실 감성 = 재진입 마찰 0).

### 10. 일시정지 (PAUSE)

| 항목 | 규칙 |
|---|---|
| 키 | `Esc` — 언제든 가능(보스전 포함). 게임 클럭·보스 타이머 정지 |

### 12. 온보딩 — 튜토리얼 텍스트 0으로 60초 안에 가르치기 ★

**제약:** 심사자는 차갑게 들어와 몇 분 안에 판단한다. 스탠스·투자·슬롯 순서는 **어떤 게임에도 없는 규칙**이라 추측 불가능하다. 그런데 t=0엔 속성 투자가 전부 0이라 **QWER이 아무 일도 안 한다** — 신호기가 스스로를 못 가르친다.
**해법: 튜토리얼 화면이 아니라 스테이지 1의 *구성*으로 가르친다.** 전부 `onboarding.json` 스크립트(코드 아님).

| 시각 | 무슨 일이 | 무엇을 가르치나 |
|---|---|---|
| **0:00–0:12** | 느린 잡몹 소수. 무속성으로 쏨 → 죽긴 죽는데 **느리다**. HUD엔 테마 뱃지(`물 70%` 등)와 상성 미니휠이 이미 떠 있음 | "기준선(×1)은 이 느낌" |
| **0:12 · Lv2** | **첫 드래프트 = 3장 전부 속성 카드로 고정**: `불+1 / 물+1 / 풀+1` (`onboardingForcedDrafts: [ "element", "element", "weapon" ]`). 각 카드에 큰 상성 화살표 글리프 | **"속성이라는 축이 존재한다"**를 우회 불가능하게 학습. 무엇을 골라도 교육적(풀→×2 쾌감 / 불→×0.5 학습 / 물→×1). 정보는 화면에 있으므로 **운빨 아님** |
| **0:12** | 확정 순간 **그 스탠스가 자동 장착**(`autoEquipFirstElement: true`) + HUD 스탠스 인디케이터의 **해당 키캡이 번쩍** | "이 글자가 이 상태다" — 키와 상태의 결속 |
| **0:12–0:20** | 같은 적을 같은 무기로 쏘는데 **숫자와 이펙트가 방금 전과 다르다** | **×2 / ×0.5를 몸으로 목격** |
| **0:20 · Lv3** | **두 번째 드래프트 = 남은 속성 2장 + 무기 1장** | 이제 속성 **2개 보유** → **QWER 전환이 의미를 갖는 최초의 순간** |
| **0:20–0:45** | **대비 웨이브**: 화면 **좌 레인 = 테마 속성 적**, **우 레인 = 테마를 이기는 속성 적**이 동시에 내려옴. 색+글리프로 명확히 구분. 죽을 수 없을 만큼 관대 | 한쪽은 녹고 한쪽은 안 녹는다 → 키를 눌러본다 → **반전** → **유레카.** (3속성이 순환이므로, 보유한 2속성이 무엇이든 두 레인의 배율은 반드시 다르다 = 어떤 테마에서도 성립) |
| 상시 | **`stanceHintPulse: true` (스테이지 1 전용, 이후 영구 OFF)** — HUD 키캡 중 *지금 화면의 적에게 ×2인 키*가 부드럽게 맥동 | 손가락이 텍스트 없이 따라간다 |
| **0:45 · Lv4** | 무기 카드 → 슬롯 2 채워짐. 투자가 `+1`이면 **슬롯 1은 속성 테두리, 슬롯 2는 회색 테두리** | **"앞의 N개만 부여"**가 그림으로 처음 드러남 |
| **~1:35** | 작은 위기 세션 (`stage1CrisisScale: 0.5`) | 리듬 학습 |
| **2:00** | **온보딩 보스 = 부위 2개짜리 아주 단순한 복합 보스** (좌 = A속성 / 우 = A를 이기는 속성 / 코어 = 노말). 부위가 크고 색이 명확. `stage1BossHpScale: 0.6` | **배운 것의 즉시 시험 = 스탠스 퍼즐 최초 체험.** 30~60초 제출 영상의 장면이 바로 여기 |

**Q(노말)는 no-op이 아니다 — 온보딩이 그것도 가르친다.** 불+4로 물 적을 때리면 `×0.5`(최악). `Q`를 누르면 `×1`(안전). **Q = 상성 회피 스탠스 = 항상 유효한 바닥.** 대비 웨이브에서 플레이어는 자연히 이것을 발견한다.

**온보딩이 쓰는 텍스트: 0줄.** 사용 채널 = ① 강제 드래프트 구성 ② 자동 스탠스 장착 ③ 색·글리프 ④ 키캡 맥동 ⑤ 좌/우 대비 배치 ⑥ 데미지 이펙트 차이.

### 13. localStorage

```
키: "nan2026.v1"            ← 반드시 접두어. 이 origin은 블로그 전체와 공유된다.
{ "v": 1,
  "best": { "normal": 0, "hard": 0, "hell": 0, "disaster": 0 },
  "runs": 0 }
```
- 읽기/쓰기 **전부 try/catch**. 실패(사파리 프라이빗·비활성) → **메모리 폴백, 게임은 정상 진행**(결과 화면에서 크래시 금지).
- `v` 불일치 → **조용히 초기화**. 오락실 감성상 기록 소멸 수용(명시적 결정).
- **점수 위변조는 방어하지 않는다** — 로컬 전용·계정 없음·"이 기계의 기록" 감성. **명시적으로 수용**하고 다시 논의하지 않는다.
- 초기화 수단 = 타이틀 하단 `기록 초기화`.

### 15. 이 섹션이 다른 섹션에 요구하는 계약 (인터페이스)

| 요구 대상 | 내용 |
|---|---|
| **HUD 섹션** | 다음이 상시 표시되어야 이 플로우가 성립: 테마 뱃지(구성비) · 페이즈/보스 타이머 · **대기 드래프트 배지** · 현재 스탠스(키캡 표기) · **슬롯 4칸의 부여 상태(부여/무속성)** · 속성 투자 3종 · HP · 코인 · 리롤/폭탄/실드/토큰 스톡 · 점수 |
| **HUD 섹션** | 모든 UI의 속성 순서 = **Q(노말) → W(불) → E(물) → R(풀)** 고정 |
| **드래프트 섹션** | §4.3 카드 표기 필수 항목 · 유효 후보 3장 미만 시 **코인 폴백 카드**(`draftFallbackCoins: 40`) |
| **적/스테이지 섹션** | 잡몹의 **화면 하단 이탈 = 소멸 + XP 영구 소실**(파밍 리스크의 실제 메커니즘) |
| **테마 추첨 섹션** | **스테이지 1은 노말 테마 제외**(`stage1ExcludesNormalTheme: true`) — 온보딩이 상성을 가르쳐야 함 |
