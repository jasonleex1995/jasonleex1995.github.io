# 적 · 중간보스 · 위기세션 · 보스 · 스테이지

> 이 절은 **콘텐츠 전량의 확정 스펙**이다. 이후 밸런싱은 이 절에 등장하는 **JSON 키의 값만** 바꾼다. 새 필드·새 규칙·새 코드가 필요하면 그건 밸런싱이 아니라 설계 변경이므로 금지.

## 0. 대원칙 — "어휘 조합, 신규 코드 0"

적·보스는 **저작물이 아니라 조합**이다. 확정된 3개 어휘만 존재하며, 모든 적/중간보스/보스 부위는 이 어휘의 조합으로 정의된다.

| 어휘 | 개수 | 확장 |
|---|---|---|
| **이동 거동** `movement_id` | 8 | **동결.** 추가 = 설계 변경 |
| **공격 거동** `attack_id` | 6 | **동결** (기존 §적 공격 거동 표와 1:1) |
| **보스 부위 타입** `part_type` | 4 | **동결** |

> **AI 생성물이 어휘 밖 값을 쓰면 로드 실패**(검증 게이트, §9). 따라서 AI가 콘텐츠를 아무리 많이 생성해도 **엔진 코드는 늘지 않는다** — 4주 솔로 예산의 근거.

## 1. 스테이지 테마

### 1.1 테마 풀 (6종) — **노말 테마 없음 (확정)**

노말 테마는 그 스테이지 동안 속성 시스템을 통째로 끄고, 직전에 속성을 투자한 플레이어를 처벌한다 → **폐기**. (v0.5의 "평원" 삭제.) 모든 테마는 물·불·풀 중 하나를 가진다.

| `theme_id` | 이름 | `element` | `intro_ok` | 정체성 (아레나·시그니처) |
|---|---|---|---|---|
| `sea` | 바다 | 물 | ✔ | 개활 수면. 느린 대형 개체, 유도 위주. **가장 읽기 쉬움** |
| `glacier` | 빙원 | 물 | ✖ | 체류형 포대 + 둔화. 라인 회피 시험 |
| `volcano` | 화산 | 불 | ✔ | 장판·분출. 공간 제한 |
| `desert` | 사막 | 불 | ✖ | 초고속 소형. **파밍 리스크의 화신** |
| `forest` | 숲 | 풀 | ✔ | 확산탄 벽. 편대 밀집 |
| `bog` | 늪 | 풀 | ✖ | 상태이상(둔화) 다발 + 시야 압박 |

**속성당 정확히 2종**. 이 대칭이 아래 추첨 보장을 성립시킨다.

### 1.2 추첨 규칙 (스테이지 1~5)

| 키 | 값 | 의미 |
|---|---|---|
| `theme_draw_count` | 5 | 6종 중 5종 |
| `theme_draw_replacement` | `false` | **런 내 테마 중복 금지** |
| `theme_stage1_requires_intro_ok` | `true` | 스테이지 1은 `intro_ok` 테마만 |
| `theme_order` | shuffle | 나머지는 완전 셔플 |

- **3속성 커버리지 자동 보장 (증명)**: 6종 중 5종 비복원 추출 = 정확히 1종만 탈락 → 각 속성은 2종 중 최대 1종만 빠짐 → **모든 런에 물·불·풀 테마가 각각 최소 1회 등장.** 즉 "물 테마 5연속으로 불+4 투자가 사망" 같은 막다른 운은 **구조적으로 불가능**하며, 시뮬로 증명할 필요조차 없다.
- **스테이지 1 배치 항상 가능 (증명)**: `intro_ok` 3종 중 최대 1종만 탈락 → 항상 ≥2종이 런에 포함.
- **스테이지 6 = 고정 최종** (`theme_id: "finale"`, 추첨 대상 아님, §8).

### 1.3 속성 혼합 규칙 — **70/10/10/10 (확정)**

테마 속성을 `T`라 할 때, 상성상 `T`를 이기는 속성을 `counter`(= 플레이어의 정답 스탠스), `T`가 이기는 속성을 `prey`라 한다.

| 구간 | 비율 키 | 값 | 정답 스탠스(`counter`)로 때릴 때 |
|---|---|---|---|
| 테마 속성 `T` | `theme_mix.theme` | **0.70** | **×2** |
| `counter` | `theme_mix.counter` | 0.10 | ×1 |
| `prey` | `theme_mix.prey` | 0.10 | **×0.5** ← 특화의 세금 |
| 노말 | `theme_mix.normal` | 0.10 | ×1 |

- 예) 화산(불): 불 70% / 물 10% / 풀 10% / 노말 10%. 정답 = **물 스탠스**(70%에 ×2), 대신 풀 10%에는 ×0.5.
- **비율의 기준 = 스테이지 잡몹 총 개체 수** (HP 총량 아님, 스폰 확률 아님).
- **롤 granularity = 웨이브 단위, 개체 단위 롤 금지** (`element_assign: "per_wave"`). 웨이브 리스트는 **빌드타임에 결정론적으로 편성**되어 스테이지 총합이 70/10/10/10에 오차 ±3%p 안으로 들어간다 → **테마 고지가 준 정보가 실제로 참**이 된다(= 정보 기반 선택 기둥이 실제로 작동).
- **혼합 비율 제외 대상**: 엘리트(§2.7), 중간보스(§4), 새떼(§5), 보스(§6) — 각자 별도 속성 규칙을 따른다.
- **테마 고지에는 이 4개 수치가 그대로 표시된다** (대략이 아님). 표현 형식은 UI 절 소관.

### 1.4 스테이지 진행 곡선 — **난이도(배속)와 완전히 별개 (확정)**

> 난이도 = 배속 하나뿐이라는 원칙은 **런 간(across runs)** 규칙이다. 아래는 **런 내(within a run)** 진행 곡선이며 모든 난이도에서 동일하다. 두 축은 곱해질 뿐 서로를 침범하지 않는다.

| 스테이지 | `enemy_hp_scale` | `spawn_density_scale` | `midboss_count` | `elite_per_wave_chance` | `rear_spawn_allowed` |
|---|---|---|---|---|---|
| 1 | 1.0 | 0.70 | 1 | 0.10 | `false` |
| 2 | 1.5 | 0.85 | 1 | 0.15 | `false` |
| 3 | 2.2 | 1.00 | 2 | 0.20 | `true` |
| 4 | 3.2 | 1.15 | 2 | 0.25 | `true` |
| 5 | 4.5 | 1.30 | 2 | 0.30 | `true` |
| 6 (최종) | 6.0 | 1.50 | 2 | 0.35 | `true` |

- **후반 = 스펀지 방지**: HP 스케일만 올리면 후반이 무르다 → **아키타입 해금**(§2.6 `unlock_stage_min`)이 함께 올라 *다른 적*이 나온다. 스테이지 1은 로스터 4종 중 2종만, 스테이지 5는 전부 + 후방 진입.
- 스테이지 로스터 교체는 **없음**(콘텐츠 물량 5배 방지). 공용 아키타입 재사용 + 테마 시그니처 1종 + 해금 스케줄로 차별화.

## 2. 적

### 2.1 이동 거동 어휘 (8종, 동결)

| `movement_id` | 이름 | 거동 | 파라미터 키 | 이 이동이 존재하는 이유(무기 시너지) |
|---|---|---|---|---|
| `dive` | 직하강 | 상단 진입 → 등속 하강 → 하단 이탈 | `speed` | 포워드/벌컨 |
| `weave` | 사인 하강 | 좌우 사인 흔들며 하강 | `speed, amp_px, freq_gs` | 확산/팬 |
| `column` | 종대 스트림 | **같은 x축에 일렬 종대**로 줄줄이 하강 | `speed, gap_gs` | **관통 랜스 "줄 세우기"** |
| `strafe` | 측면 횡단 | 좌/우 벽 진입 → 수평 횡단 → 반대편 이탈 | `speed, y_px` | **부메랑 "라인 정렬"**, 후방/전방위 |
| `anchor` | 진입 체류 | 상단 진입 → `y_hold_px`에 정지 → 좌우 소폭 왕복 | `enter_speed, y_hold_px, sway_amp_px, leave_after_gs` | 드론/옵션, 관통 저격 |
| `orbit_drift` | 선회 접근 | 플레이어 쪽으로 호를 그리며 접근 → `keep_dist_px` 유지 | `speed, turn_rate_deg_gs, keep_dist_px` | **오빗/오라 "파고들어 몸빵"** |
| `charge` | 돌진 | 상단 체류 → 텔레그래프 → 플레이어 방향 1회 직선 돌진 → 이탈 | `wind_up_gs, dash_speed` | 위치 판단 시험 |
| `rear_in` | 후방 진입 | **하단 밖에서 상승 진입** | `speed, warn_gs` | **후방/전방위 무기의 존재 이유** |

- `rear_in`은 `rear_spawn_allowed: true`(스테이지 3+)에서만, 그리고 **반드시 `warn_gs` (0.8 게임초) 동안 하단 경계에 진입 표식**이 선행한다. 무-트위치 기둥 준수. 뒤에서 소리 없이 나오는 적은 금지.
- `charge`의 `wind_up_gs` ≥ `telegraph_min_gs`.

### 2.2 공격 거동 어휘 (6종, 동결) — 이미터 파라미터 + 텔레그래프

| `attack_id` | 파라미터 키 | `telegraph_gs` 하한 | 텔레그래프 형태 |
|---|---|---|---|
| `straight` | `count, speed, interval_gs` | `telegraph_min_gs` = **0.8** | 발사구 점멸 |
| `fan` | `count, arc_deg, speed, interval_gs` | 0.8 | 부채꼴 호 프리뷰 |
| `homing` | `count, speed, turn_rate_deg_gs, lifetime_gs, interval_gs` | 0.8 | 락온 마크 |
| `laser` | `charge_gs, width_px, duration_gs, interval_gs` | `telegraph_min_gs_laser` = **1.2** | 라인 프리뷰 |
| `zone` | `radius_px, delay_gs, duration_gs, interval_gs` | `telegraph_min_gs_zone` = **1.0** | 바닥 원 |
| `status` | `count, speed, effect, duration_gs, interval_gs` | 1.0 | 크고 느린 전용 탄 |
| `none` | — | — | 사격 안 함 (물량·몸통 전용) |

- `status.effect ∈ { slow, stun }`. `slow`가 기본. **`stun`은 `status_stun_min_difficulty: "hard"` 이상 + `status_stun_max_per_stage: 2`(스테이지당 최대 2개체만 보유)** — 문서의 잠금 결정 유지.
- **탄막 불변조건 (시뮬이 인증, §9)**: `bullet_speed_max_px_gs`, `bullet_gap_min_px`(통과 가능 최소 틈), `telegraph_concurrent_max: 2`(§6.5).

### 2.3 HP 밴드

`hp = enemy_hp_base × band.hp_mult × enemy_hp_scale[stage]`

| `band` | `hp_mult` | 크기 | `coin_drop_chance` | 역할 |
|---|---|---|---|---|
| `chaff` | 1.0 | 소 | **0.0** | 1~2히트, XP 사료 |
| `line` | 2.5 | 소중 | **0.0** | 편대 구성원 |
| `turret` | 6.0 | 중 | **0.15** | 화면 체류, 압박 유지 |
| `bruiser` | 12.0 | 대 | **0.15** | DPS 체크 |

> **코인 드랍 주체 확정**: `turret`·`bruiser` 밴드(확률) + 엘리트(확정) + 중간보스(확정) + 보스(확정). `chaff`·`line`은 **코인 0**. 이것이 v0.5의 "일부 잡몹만"의 정의다.

### 2.6 테마별 로스터 — **테마당 정확히 4종** (공용 3 + 시그니처 1)

각 엔트리는 `unlock_stage_min`을 가진다 = 그 스테이지 인덱스 미만에서는 스폰되지 않음 → **같은 테마라도 스테이지 2에서 만나면 2종, 스테이지 5에서 만나면 4종.**

| 테마 | 로스터 (`unlock_stage_min`) |
|---|---|
| **바다** `sea` | `drifter`(1) · `spitter`(1) · `stalker`(3) · **`siren_ray`**(2) |
| **빙원** `glacier` | `spitter`(1) · `turret_pod`(2) · `column_ant`(3) · **`frost_lance`**(2) |
| **화산** `volcano` | `spitter`(1) · `mortar_hulk`(3) · `rear_dart`(3) · **`magma_bomb`**(2) |
| **사막** `desert` | `drifter`(1) · `column_ant`(1) · `rear_dart`(3) · **`dust_runner`**(1) |
| **숲** `forest` | `spitter`(1) · `flanker`(2) · `hexer`(3) · **`thorn_weaver`**(1) |
| **늪** `bog` | `hexer`(1) · `flanker`(2) · `turret_pod`(3) · **`bog_hexer`**(2) |

> **9 공용 + 6 시그니처 = 15 아키타입**이지만 전부 (이동 × 공격 × 밴드) 조합이므로 **엔진 코드 추가 0**. 이것이 4주 예산의 핵심.

### 2.7 엘리트 — **정의 확정: 별도 개체가 아니라 접두(prefix)**

**엘리트 = 웨이브 안의 일반 아키타입 1기에 붙는 강화 플래그.** 이동·공격 거동은 원본 그대로.

| 키 | 값 | |
|---|---|---|
| `elite_per_wave_max` | 1 | 웨이브당 최대 1기 |
| `elite_per_wave_chance` | 스테이지별(§1.4) | 시드 스트림 `rng.spawn` |
| `elite_hp_mult` | 4.0 | |
| `elite_size_mult` | 1.4 | |
| `elite_xp_mult` | 6.0 | |
| `elite_coin` | 3 | **확정 드랍** |
| `elite_heal_drop_chance` | 0.12 | ← v0.5 "드물게 엘리트 드랍"의 정의 |
| `elite_band_allowed` | `["line","turret","bruiser"]` | `chaff` 엘리트 금지(허수아비) |
| `elite_element_allowed` | `["물","불","풀"]` | **노말 엘리트 금지** — 상성 재미가 0이므로 |

**엘리트 ≠ 중간보스**: 엘리트는 웨이브의 일부(이탈 타이머 없음, 화면을 지나감). 중간보스는 독립 이벤트 개체(§4).
엘리트는 **반드시 군중 속에서 즉시 식별 가능한 표식**을 가져야 한다(형태 규칙은 렌더 절 소관). 표식 없는 엘리트 = 코인·회복 수급이 랜덤으로 체감 = 상점 경제 붕괴.

### 2.8 스폰 · 편대 · 상한

**편대 어휘 `formation_id` (6종, 동결)**

| `formation_id` | 배치 | 전용 |
|---|---|---|

## 3. 잡몹 페이즈 (게임시간 2분) — **파밍 = 진짜 선택**

### 3.1 종료 조건 (확정)

| 키 | 값 |
|---|---|
| `mob_phase_duration_gs` | **120** — 고정 게임시간 하드 타이머 |
| `wave_interval_gs` | 9.0 — 기본 웨이브 간격 |
| `wave_clear_advance` | **`true`** — **현재 웨이브를 전멸시키면 다음 웨이브 즉시 소환** |
| `mob_phase_max_waves` | 14 — 도달 시 추가 스폰 정지(무한 파밍 방지) |
| `mob_phase_exit_fade_gs` | 3.0 — 종료 시 잔존 잡몹 페이드아웃 소멸 |
| `mob_phase_exit_clear_bullets` | `true` — 잔존 적 탄 소거(보스 시작을 공정하게) |
| `phase_end_autocollect` | `true` — 미수집 XP·코인은 페이즈 종료 시 반경 무시하고 **전부 자동 수집** (무-노가다 기둥) |
| `mob_phase_skip` | **`false`** — 보스로 조기 진입 불가 |

### 3.2 리스크가 **무엇인지** (메커니즘, 선언 아님)

| 리스크 장치 | 규칙 | 만드는 선택 |
|---|---|---|
| **① 이탈 = XP 영구 소멸** | 화면 밖으로 나간 적의 XP·코인은 **영구 소멸**(`enemy_exit_forfeits_reward: true`). 부활·재진입 없음 | 잡으려면 **위로 올라가 요격**해야 함 → 상단 = 반응 여유 최소 = **위치 리스크**. 트위치가 아니라 "얼마나 앞에 설 것인가"의 각오 |
| **② 전멸 = 즉시 다음 웨이브** | `wave_clear_advance` | **플레이어가 자기 화면 밀도를 스스로 올린다.** 공격적 = 2분에 14웨이브 = XP 최대 + 화면 최대. 소극적 = 2분에 ~13웨이브 스폰 중 절반이 그냥 지나감 = XP 최소 + 안전. 이게 리스크/리워드의 본체 |
| **③ 고XP 도망자** | `dust_runner` (XP ×5, 초고속 `dive`) | 등장 즉시 위로 뛰어야 잡힘. 안 잡으면 2초 후 소멸 |
| **④ 중간보스 병행 압박** | 중간보스 교전 중에도 웨이브는 계속 온다 | 큰 XP vs 이중 압박 |

> **이 구조가 "잘 골랐나"를 3분 뒤 보스 타이머로 채점한다.** 파밍 부족 = 언더레벨 = 타이머 실패. 파밍 과다 = 잡몹 페이즈에서 죽음. 둘 다 *선택의 결과*이지 운이 아니다.

### 4.1 정의

- **단일 몸체. 부위 없음** (복합은 스테이지 보스만). 자체 아키타입 3종, 전 테마 공유.

### 4.3 "선택적"이 무슨 뜻인지 (확정)

**중간보스는 등장 후 `midboss_leave_after_gs`(30 게임초)가 지나면 화면 위로 이탈해 사라진다.** 이탈 = 보상 0.

- 무시해도 **쫓아오지 않는다**(추격 금지 — 그러면 사실상 강제가 되어 "선택"이 거짓이 됨).
- 다만 **이탈 전까지는 계속 사격한다** → 무시 = "30초간 추가 탄막을 회피하며 웨이브를 파밍한다"는 선택. 공짜 회피가 아니다.
- → **진짜 선택**: 잡는다(큰 XP + 코인 + 회복 확률, 대신 30초를 통째로 씀) vs 무시한다(30초를 웨이브 파밍에 쓰되 탄막은 그대로).

### 4.4 속성 규칙 — 보스전의 예고편

**중간보스 속성 = 테마 속성도 노말도 아닌 것** (`counter` 또는 `prey`).
→ 테마 정답 스탠스(예: 화산에서 물)를 켜둔 채로는 중간보스가 잘 안 죽는다 → **스탠스를 바꾸게 만든다.** 스테이지 보스(§6)가 요구할 스탠스 전환을 잡몹 페이즈에서 미리 연습시키는 장치.

## 5. 위기 세션 (새떼) — **화력 페이오프의 순간**

### 5.1 정체성

**새떼는 회피 시험이 아니라 화력 시험이다.** 60기가 화면을 뒤덮고 밀려온다. 화력이 있으면 **한 화면 전체가 동시에 터진다** — 런에서 가장 짜릿한 25초. 화력이 없으면 화면이 막히고 아래로 밀려나며 XP 덩어리를 통째로 놓친다.

### 5.2 규칙

| 키 | 값 |
|---|---|
| `crisis_per_stage` | **1** (매 스테이지 필수) |
| `crisis_start_gs` | **95** (잡몹 페이즈 120초 중 마지막 25초) |
| `crisis_duration_gs` | **25** |
| `crisis_warning_gs` | **3.0** — 시작 3 게임초 전 예고 배너 + 스팅 (텔레그래프 기둥: 기습 금지) |
| `crisis_suspends_waves` | **`true`** — 새떼 중 일반 웨이브 스폰 정지 (밀도 폭발 방지 = 관대함) |
| `crisis_total` | **60** × `swarm_total_scale[stage]` |
| `crisis_sub_waves` | 6 (10기 × 6파, `arc`/`v_wedge` 편대) |
| `crisis_fail_condition` | **없음** |

### 5.3 구성 — 전용 아키타입 2종

| `archetype_id` | 비율 | `band` | `movement_id` | `attack_id` | |
|---|---|---|---|---|---|
| `swarm_chaff` | 0.90 | `chaff` (HP ×0.5) | `weave`(빠름) | **`none`** | **사격 안 함.** 위협 = 물량·시야·공간 |
| `swarm_lancer` | 0.10 | `chaff` | `dive` | `straight`(1발/2.5초, 저속) | 소수의 탄으로 "완전 무해"는 아니게 |

### 5.4 속성 — **테마 속성 100% (혼합 규칙의 명시적 예외)** ★

`crisis_element_rule: "theme_pure"`

**이것이 새떼 설계의 핵심.** 60기 전부가 테마 속성 → **정답 스탠스에 투자한 플레이어는 25초 동안 ×2로 전부 갈아버린다.** 투자가 화면 가득한 폭발로 되돌아오는 순간 = "선택이 만든 양학"이 가장 크게 보이는 장면 = 제출 영상의 클라이맥스.
반대로 투자를 안 한 플레이어는 ×1로 밀려난다. **처벌이 아니라 정보**다 — 다음 레벨업에서 무엇을 골라야 하는지 몸으로 알려준다.

### 5.5 실패해도 죽지 않는다 (관대함 기둥)

- `crisis_duration_gs` 종료 시 잔존 새떼는 **전원 소멸**. 사망 조건 아님.
- **회피 가능**: 하단으로 물러나 지나가길 기다릴 수 있다 → XP 포기가 유일한 대가.
- 보상: `swarm_xp` = `chaff` XP × 0.5. 60기 → **스테이지 총 XP의 ~25%**. 큰 덩어리이므로 놓치면 아프지만 런이 끝나지는 않는다.

### 5.6 "노바/오라를 못 뽑은 빌드는?" — 막다른 운 없음 (확정)

새떼의 정답은 3개다:
1. **노바/화면청소** — 설계상 캡스톤 정답
2. **오라/필드** — 밀집을 학살
3. **폭탄** — 잡몹에 광역 즉사급 + 탄 소거 → **새떼에 완전 유효**. 상점에서 코인으로 항상 살 수 있음

→ 무기 드래프트 운과 무관하게 **폭탄이라는 보편 해답이 항상 존재**한다. 새떼는 "특정 무기를 못 뽑으면 죽는 시험지"가 아니라 "화력을 준비했는가"의 시험지다.

## 6. 복합 보스 규칙

> **모든 스테이지 보스는 복합.** 중간보스는 단일. 예외 없음.

### 6.1 구조

| 키 | 값 |
|---|---|
| `boss_part_count` | **4** (최종만 5, §8) |
| `boss_part_regen` | **`false`** — 부위 파괴는 영구 |
| `boss_summons_allowed` | **`false`** ← 스테이지 보스는 잡몹을 소환하지 않는다 |
| `boss_part_hit_priority` | `"outermost_first"` — 히트박스가 겹치면 바깥 부위 우선. 부위 파괴 시 그 히트박스가 제거되어 **자동으로 코어가 노출**됨(별도 규칙 불필요) |
| `part_hp_share` | 부위별 비중, **합계 = 1.0 강제**(±0.001). `boss_hp_total`에 곱해짐 |

### 6.2 부위 타입 어휘 (4종, 동결) — 애드혹 보스 금지

| `part_type` | 파괴 시 효과 (**규칙이며, 보스별 특수 코드 아님**) | 필수 |
|---|---|---|
| `mobility` | 보스 이동 속도 × `part_mobility_penalty`(**0.5**), 회피 기동 패턴 중단 | 선택 |
| `armament` | **그 부위가 담당하던 `attack_id` 1종이 영구 삭제** | 선택 |
| `armor` | 코어 피해 게이트 1단계 해제 (§6.3) | 1~2개 |
| `core` | **보스 사망** | 정확히 1개 |

이 4개가 전부다. 6개 보스가 전부 이 조합이므로 **보스별 특수 코드 = 0**.

### 6.3 ★ 코어 소프트 게이트 (설계의 급소) ★

> **질문**: 부위를 무시하고 코어만 때려 죽일 수 있는가?
> **답**: 때릴 수는 있다. 하지만 산술적으로 3분 안에 못 죽인다.

```
코어_피해배율 = part_armor_core_mult ^ (살아있는 armor 부위 수)
part_armor_core_mult = 0.4
```

| 살아있는 `armor` | 코어 피해 배율 |
|---|---|
| 2 | **×0.16** |
| 1 | ×0.40 |
| 0 | **×1.00** |

- **강제 게이팅이 아니라 배율** → "코어 직행" 자유는 존재하되 명백히 손해. **하드 게이트가 아닌 소프트 게이트**이므로 플레이어가 규칙을 몰라도 벽에 부딪히지 않고 *느려짐으로* 배운다(정보 전달이 곧 피드백).
- **`core`의 속성 = 항상 노말 (`boss_core_element: "노말"`, 고정)** → 어떤 스탠스로도 ×1 마무리 가능 = **절대 브릭 없음**. 문서의 "노말이 안전 바닥" 원칙과 정확히 일치.
- **`mobility`·`armament` 파괴는 순수 선택** — 코어 배율에 영향 없음. 대신 회피가 쉬워진다.
  → **3분 타이머 하의 진짜 트레이드오프**: *"시간을 써서 무장을 깨고 안전하게 갈 것인가, `armor`만 깨고 코어로 직행할 것인가."* 이것이 보스전의 결정.

### 6.4 부위 속성 배정 규칙 (AI 하드 제약)

| # | 규칙 | 키 | 이유 |
|---|---|---|---|
| **R1** | `core` 속성 = **항상 노말** | `boss_core_element` | 브릭 방지 |
| **R2** | 주변부 속성에 **노말 금지** | `boss_part_normal_forbidden: true` | 모든 주변부는 상성 대상 |
| **R3** | 주변부는 **서로 다른 속성 ≥ 2종** | `boss_part_element_distinct_min: 2` | **스탠스 1개 샌드백 금지** |
| **R4** | **테마 속성은 최대 1개 부위** | `boss_part_theme_element_max: 1` | 보스 = 테마의 반복이 아니라 **커버리지 시험** |
| **R5** | **`armor` 부위 속성 ≠ 테마 속성** | `boss_armor_element_not_theme: true` | 가장 오래 때리는 부위가 **다른 스탠스를 강제** |
| **R6** | `armor` 부위 수 ∈ {1, 2} | `boss_armor_part_count_range: [1,2]` | 게이트 강도 |

> R4+R5의 결합이 이 게임의 셀링 포인트를 **규칙으로** 보장한다: 테마 정답 스탠스 하나만 갈고 온 특화 빌드는 **보스에서 반드시 스탠스를 바꿔야 한다.** 6개 보스를 AI가 생성해도 전부 스탠스 퍼즐이 됨이 보장된다.

### 6.5 페이즈 · 타이머 · 텔레그래프 예산

| 키 | 값 | |
|---|---|---|
| `boss_intro_gs` | **2.5** | 등장 연출 중 보스 무적 |
| `boss_timer_starts_after_intro` | **`true`** | 연출은 3분에서 차감 안 됨 |
| `boss_phase_thresholds` | **`[0.6, 0.3]`** | **코어 HP** 기준 3페이즈 |
| `boss_phase_transition_gs` | **1.5** | 전환 중 보스 무적 + 연출 = **공짜 숨돌릴 틈**(관대함) |
| `boss_timer_pauses_on_phase_transition` | **`true`** | 전환 시간은 3분에서 차감 안 됨 = 공정 |
| `boss_timer_expire` | **즉사** | 서서히 죽는 연출 없음. 오락실 룰 |

- 페이즈 = **코어 HP 임계만** 사용(부위 파괴와 무관) → 규칙 하나, 조합 폭발 없음. 부위 파괴 효과는 페이즈와 독립적으로 즉시 발동.
- 페이즈마다 각 부위의 `pattern_set` 인덱스가 교체된다(`interval_gs` 단축, `count`/`arc_deg` 증가). **새 `attack_id`가 생기지는 않는다.**

**텔레그래프 예산 (관대함의 정량 정의)**

| 키 | 값 |
|---|---|
| `telegraph_min_gs` | 0.8 |
| `telegraph_min_gs_laser` | 1.2 |
| `telegraph_min_gs_boss_burst` | **1.5** |
| **`telegraph_concurrent_max`** | **2** |
| `bullet_speed_max_px_gs` | 시뮬 인증 대상 |
| `bullet_gap_min_px` | 시뮬 인증 대상 (통과 가능 최소 틈) |

> **`telegraph_concurrent_max: 2`가 보스 스크립트의 가장 강한 제약이다.** 동시에 3개를 읽으라고 요구하는 순간 그건 트위치다. AI가 생성한 부위 스크립트는 **어느 시점에도 동시 텔레그래프가 2개를 넘지 않아야** 하며, 초과하면 로드 실패(§9).
>
> **텔레그래프 예산은 Normal(×1)의 실제시간 기준으로 정의된다.** 배속은 이 여유를 갉아먹으며, 그것이 곧 상위 난이도의 정의다 — 문서의 "난이도는 옵트인(무-트위치 기둥은 Normal에서 보존)"과 일치. 실제시간 하한 같은 배속 예외는 **두지 않는다**(동일 배속 원칙 + 시뮬 결정론 유지).

> ★ **log.md의 옵션 "보스 처치 시 그 테마 무기가 드래프트 풀에 해금" = 폐기 (확정).** 이유: 드래프트 풀이 스테이지마다 변하면 "정보 기반 선택"이 아니라 예측 불가가 되고, 드래프트 가중치·시뮬 전제·UI가 전부 복잡해진다. **모든 무기는 런 시작부터 풀에 있다.**

## 7. 보스 예시 3종 (규칙의 실증)

### 7.1 「강철 가오리 (Manta)」 — 바다(물) 테마

| 부위 | `part_type` | 속성 | `hp_share` | **정답 스탠스** | 파괴 효과 |
|---|---|---|---|---|---|
| 스러스터 | `mobility` | **물** | 0.10 | 풀 | 이동 ×0.5, 좌우 회피 기동 중단 |
| 좌 지느러미 | `armor` | 불 | 0.20 | **물** | 코어 게이트 1/2 해제 |
| 우 지느러미 | `armor` | 풀 | 0.20 | **불** | 코어 게이트 2/2 해제 |
| 코어(몸체) | `core` | 노말 | 0.50 | 아무거나 | **사망** |

- 규칙 검증: R3 distinct=3 ✔ / R4 테마(물) 부위 1개 ✔ / R5 armor 둘 다 ≠ 물 ✔ / R6 armor=2 ✔ / `hp_share` 합 1.00 ✔
- **전술**: armor 2개 → 코어 ×0.16 → 반드시 깨야 함. **물 스탠스로 좌 지느러미 → 불 스탠스로 우 지느러미 → 게이트 해제 → 코어(노말) 마무리.** 스러스터(물)는 선택 — 풀 스탠스로 깨면 보스가 느려져 남은 시간이 편해지지만 그만큼 타이머를 씀.
- **커버리지 시험이 작동함**: 바다 테마를 물+4로 밀고 온 특화 빌드는 좌 지느러미(불)만 빠르게 깨고 **우 지느러미(풀)에서 막힌다** → 정확히 의도한 "특화의 대가".

### 7.2 「용광로 거신 (Kiln)」 — 화산(불) 테마

| 부위 | `part_type` | 속성 | `hp_share` | **정답 스탠스** | 파괴 효과 |
|---|---|---|---|---|---|
| 좌 포탑 | `armament` | **불** | 0.15 | 물 | `zone`(용암 장판) 영구 삭제 |
| 우 포탑 | `armament` | 풀 | 0.15 | 불 | `laser` 영구 삭제 |
| 흉갑 | `armor` | 물 | 0.25 | **풀** | 코어 게이트 해제 (armor 1개 → ×0.4 → ×1.0) |
| 노심 | `core` | 노말 | 0.45 | 아무거나 | **사망** |

- 규칙 검증: R3 distinct=3 ✔ / R4 테마(불) 1개 ✔ / R5 흉갑(물) ≠ 불 ✔ / R6 armor=1 ✔ / 합 1.00 ✔
- **전술**: armor 1개 → 코어 ×0.4 → **"흉갑 무시하고 노심 직행"이 실제로 성립하는 유일한 보스.** 흉갑(물)을 깨려면 풀 스탠스가 필요 → 풀 투자가 없으면 ×1로 흉갑을 때리느니 ×0.4로 노심을 미는 게 나을 수 있다 = **빌드에 따라 최적해가 갈리는 보스.** 포탑 2개는 순수 선택(깨면 탄이 절반).

### 7.3 「테트라크 (TETRARCH)」 — 스테이지 6 최종 ★ 커버리지 졸업 시험

| 부위 | `part_type` | 속성 | `hp_share` | **정답 스탠스** | 파괴 효과 |
|---|---|---|---|---|---|
| 물의 관 | `armor` | 물 | 0.18 | **풀** | 게이트 1/3 |
| 불의 관 | `armor` | 불 | 0.18 | **물** | 게이트 2/3 |
| 풀의 관 | `armor` | 풀 | 0.18 | **불** | 게이트 3/3 |
| 왕좌(포대) | `armament` | **노말** | 0.11 | 없음(항상 ×1) | `homing` 전탄 영구 삭제 |
| 핵 | `core` | 노말 | 0.35 | 아무거나 | **사망** |

| 최종 전용 키 | 값 |
|---|---|
| `finale_part_count` | **5** |
| `finale_armor_part_count` | **3** |
| `finale_exempt_rules` | `["R4"]` — 최종은 테마가 없으므로 R4 면제 |
| `finale_allow_normal_peripheral` | **`true`** — 왕좌만 R2 면제 |

- **armor 3개 → 코어 ×0.4³ = ×0.064.** 코어 직행은 완전히 불가능 → **3속성 전부를 반드시 깨야 한다 = 3속성 투자의 졸업 시험.** 문서가 최종에 부여한 의도가 규칙으로 강제된다.
- **왕좌(노말)** = 상성이 통하지 않는 유일한 주변부 → 어떤 스탠스든 ×1 → **"항상 ×1인 안전 바닥"의 존재 증명 부위.** 유도탄이 괴로우면 시간을 써서 깬다.
- **고유 연출 (규칙)**: 관이 하나 파괴될 때마다 아레나 배경이 그 속성의 색을 잃는다 → 3개 다 깨지면 무채색 아레나에서 핵과 단둘이 = 시각적 클라이맥스. (구체 렌더는 렌더 절 소관, 여기서는 트리거만 확정.)
- **특화 빌드도 클리어 가능 (막다른 운 없음의 수치 정의)**: 불+4 단일 특화는 풀의 관에 ×2, 나머지 두 관에는 무속성 ×1(불 스탠스로 ×0.5를 맞느니 노말 스탠스가 낫다). `finale_hp_total`은 **"단일 속성 +4 특화 빌드가 두 관을 ×1로 밀어도 3분 안에 격파 가능"**한 값으로 시뮬 튜닝한다 → **특화 = 아슬아슬하게 성공, 분산 = 여유롭게 성공.** 브릭 아님, 대가 있음.

### 7.4 나머지 보스 3종

빙원·사막·숲·늪 테마 중 실제 런에 등장하는 것들의 보스는 **§6.4의 R1~R6과 §6.7 스키마만으로 AI가 생성**한다(테마당 1종, 총 6종). 손으로 쓰는 건 위 3종뿐이며, 나머지는 규칙의 인스턴스다.

### 9.2 하드 제약 (위반 = 생성물 폐기, 재생성)

| # | 제약 |
|---|---|
| G1 | `movement_id`·`attack_id`·`part_type`·`formation_id`가 **동결 어휘 밖이면 실패**. AI는 새 거동을 발명할 수 없다 |
| G2 | **아키타입 겹침 금지** — `(movement_id, attack_id)` 쌍이 중복되면 실패(`band`가 다르면 허용). 무기 §특색 원칙과 동일한 규율을 적에게 적용 |
| G3 | 보스 부위 규칙 **R1~R6 전부** 통과 (§6.4). `part_hp_share` 합 = 1.0 ±0.001 |
| G4 | 모든 `attack_params.telegraph_gs` ≥ 해당 `attack_id`의 하한 (§2.2) |
| G5 | **어느 시점에도 동시 텔레그래프 ≤ `telegraph_concurrent_max`(2)** — 보스 `pattern_set`을 3페이즈 전부 전개해 정적 검사 |
| G6 | 모든 탄: `speed ≤ bullet_speed_max_px_gs`, 동시 전개 시 통과 틈 ≥ `bullet_gap_min_px` |
| G7 | 웨이브 리스트의 **스테이지 총 개체 수 기준 속성 비율이 `theme_mix`에 ±3%p** (§1.3). 엘리트/중간보스/새떼/보스 제외 |
| G8 | `crisis_element_rule` = 테마 100%(최종만 로테이션). 새떼에 `swarm_*` 외 아키타입 금지 |
| G9 | `rear_in` / `spawn_edge: "bottom"` 은 `rear_spawn_allowed[stage]`일 때만 |
| G10 | 모든 난수는 **명명된 시드 스트림**만 사용: `rng.theme_draw` / `rng.spawn` / `rng.elite` / `rng.drop`. 스트림 간 공유 금지(콘텐츠가 바뀌어도 이전 시뮬 인증이 무효화되지 않게) |

### 9.3 시뮬 인증 게이트 (이 절 콘텐츠의 합격 기준)

봇 3정책 × **각 보스 1000판** (Normal, 파밍 정책 = `full_farm`):

| 지표 | 키 | 임계 |
|---|---|---|
| 3속성 분산 빌드(2-1-1)의 3분 내 격파율 | `sim_gate.balanced_clear_rate` | **≥ 0.95** |
| 단일 속성 +4 특화 빌드의 격파율 | `sim_gate.specialist_clear_rate` | **≥ 0.85** ← 특화는 아슬아슬 |
| 속성 무투자 빌드의 격파율 | `sim_gate.no_element_clear_rate` | **≥ 0.55** ← 브릭 없음의 하한 |
| 단일 아키타입이 차지하는 플레이어 사망 원인 비중 | `sim_gate.archetype_lethality_max` | **≤ 0.25** ← 지배 적 없음 |
| 테마별 격파율 표준편차 | `sim_gate.theme_clear_stddev_max` | **≤ 0.06** ← 지배 테마 없음 |
| 새떼 완전 처치율 (노바/오라 없는 빌드 + 폭탄 1개) | `sim_gate.crisis_clear_without_capstone` | **≥ 0.80** ← 막다른 운 없음 |
| 잡몹 페이즈 XP: 최대 파밍 ÷ 최소 파밍 | `sim_gate.farm_xp_ratio` | **≥ 2.0** ← 파밍이 진짜 선택인지의 증명 |

> 마지막 지표가 핵심이다. **비율이 2.0에 못 미치면 "파밍 = 리스크/리워드"는 말뿐이다.** `wave_clear_advance` + 이탈 시 XP 소멸(§3.2)이 이 비율을 만드는 메커니즘이며, 시뮬이 이를 수치로 증명한 결과가 공모전 "AI 활용 기술 문서"의 근거가 된다.
