# 플레이어 · 조작 · HUD

> 이 섹션은 **모델(설계)을 확정**하고, 모든 수치는 `data/tuning.json`의 **명명된 키**로만 존재한다. 이후 밸런싱은 이 표의 값만 바꾼다 — 구조는 바뀌지 않는다.

---

## 0. 좌표계 전제

모든 공간 수치는 **논리 px** 기준이며, 실제 화면에는 letterbox 스케일로 그려진다.

| 항목 | 값 | 비고 |
|---|---|---|
| 논리 캔버스 | **1280 × 720** (16:9 고정) | 심사자 노트북 = 16:9 → 크롭·레터박스 없이 정확히 채움 |
| 플레이필드(아레나) | **560 × 720** (중앙, x=360~920) | 세로 슈팅 비율 7:9 |
| 좌 패널 | 360 × 720 (x=0~360) | 빌드/스탠스 |
| 우 패널 | 360 × 720 (x=920~1280) | 자원/점수 |

- **HUD는 플레이필드를 침범하지 않는다** — 단, 아래 3.3의 얇은 오버레이 띠 3개만 예외(반투명, 플레이어 이동 영역 밖).
- 세로 슈팅을 가로 화면에 올릴 때 남는 좌우 공간을 **패널로 쓴다** = 탄 가독성(기둥)과 정보량을 동시에 확보. 오버레이 HUD 방식은 폐기.

## 1. 플레이어 기본 스펙

### 1.1 체력 모델 — **수치형 HP (정수), 세그먼트 표시**

- HP는 **연속 수치**. 하트/라이프제 아님. 적 탄은 **탄마다 데미지가 다르다**(적 데이터 스키마의 `damage` 필드).
- HUD에는 **세그먼트 바**로 그린다(`hpMax / hpSegment` = 5칸) → 수치형의 튜닝 자유도 + 이산형의 한눈 판독을 동시에.
- **잔기(라이프) 개념 없음.** HP 0 = 즉시 사망 = 런 종료(컨티뉴는 별도 섹션의 1회 하드캡 규칙).

**피격 데미지 공식 (구조 확정 — 항 추가/제거 금지)**

```
taken = ceil( max( bulletDamage - defense , bulletDamage * damageFloorRatio ) )
hp   -= taken
```

- **방어력 = 정액 감산**, 하한은 **원본의 25%** (`damageFloorRatio`). 정액이라 잡탄 칩딜은 거의 무해해지고(관대 기둥), 큰 탄·레이저는 여전히 아프다(위치잡기 기둥). 하한이 있어 무적화 불가.
- 데미지 난수 없음. 크리티컬 없음. (플레이어→적 데미지 공식은 별도 섹션.)

**적 탄 데미지 티어 (기본값)**

| 티어 | 키 | 값 | 예 |
|---|---|---|---|
| 소형 | `enemyBullet.damageSmall` | 8 | 직선탄/확산탄 |
| 중형 | `enemyBullet.damageMedium` | 14 | 유도탄, 보스 부채꼴 |
| 대형 | `enemyBullet.damageLarge` | 22 | 대형탄 |
| 레이저 | `enemyBullet.damageLaser` | 22 | 차지 후 발사 |
| 장판 틱 | `enemyBullet.damageZoneTick` | 10 | i-frame 공유 → 초당 최대 1회 |
| 상태이상 탄 | `enemyBullet.damageStatus` | 6 | 데미지는 낮고 효과가 본체 |

→ `hpMax=100`, i-frame 1초 → **죽으려면 최소 5~12초 동안 연속으로 맞아야 한다.** 즉사·순삭 불가 = 관대 기둥의 산술적 보증.

**저체력 상태**: `hp / hpMax <= lowHpThreshold(0.30)` → 화면 가장자리 붉은 비네트 1Hz 펄스 + HP 바 펄스 + 심박 SFX.

### 1.2 이동 모델 — **관성 없음(즉시)**

- `velocity = dir * moveSpeed`. **가속/감속 없음, 즉시 정지.** 관성은 미세 조작 실패를 만들고 그건 곧 트위치 → 기둥 위반.
- 단, 코드에는 **지수 스무딩 항이 존재**하고 `moveResponseTau = 0`(초)이 기본이다. 나중에 "살짝 미끄럽게"가 필요해도 **숫자만** 바꾸면 된다(설계 변경 아님).
- **대각선 정규화 필수** (`diagonalNormalize = true`). 정규화 없으면 대각 이동이 공짜로 41% 빨라져 회피 공간 밸런스 전체가 무의미해진다.
- **이동 영역**: 아레나 rect에서 상 56 / 하 56 / 좌우 20 inset → **520 × 608**. 상하 이동 자유(복합 보스 부위 조준 = 위치잡기 기둥의 전제).
- 적/보스와 겹쳐도 밀리지 않는다(`enemy.contactPush = false`). 관통 통과.

### 1.3 히트박스 — **작고, 항상 보인다**

| 대상 | 모델 | 값 |
|---|---|---|
| 플레이어 스프라이트 | 원 | 반지름 14 (지름 28) |
| **플레이어 히트박스** | 원 | **반지름 4** (스프라이트의 **29%**) |
| 적 탄 히트박스 | 원 | 시각 반지름 × **0.8** (`enemyBullet.hitboxScale`) |
| 적 기체 히트박스 | 원 | 적 스키마의 `hitboxRadius` |

- **히트박스는 상시 렌더**(`hitboxAlwaysVisible = true`). 보이지 않는 관대함은 관대함이 아니다 — 플레이어가 스침을 피격으로 오독하면 기둥이 무너진다.
- **시각 규격 (★ 스탠스 판독의 핵심)**: 흰색 코어 도트(r=4) + 그 둘레의 **현재 스탠스 색 링**(r=6, 2px). 눈이 이미 가 있는 곳에서 스탠스를 읽는다.
- **그레이즈 없음 (확정).** 그레이즈는 탄에 붙어 있을수록 이득 = 트위치·탐욕 유도 → 기둥 위반. 관련 키·UI·점수 항목 모두 없음.

### 1.4 피격 처리 — **무적시간 존재 (확정)**

| 규칙 | 값 |
|---|---|
| 무적시간 | `iFrameDuration = 1.0` **게임초** |
| 시각 | 기체 10Hz 깜빡임(alpha 1↔0.35) + 흰 아웃라인 1회 펄스. 히트박스 링은 **계속 선명하게 유지**(깜빡여도 위치는 항상 읽힘) |
| 넉백 | **없음** (`knockback = 0`) |
| 이동 경직 | **없음** (`hitstunDuration = 0`) |
| 피격 시 탄 소거 | **없음** (`clearBulletsOnHit = false`) — 소거는 폭탄의 정체성이므로 침범 금지 |

- **모든 피해원이 i-frame을 공유한다** — 적 탄 / 몸통 충돌 / 설치 장판 / 레이저. 그러므로 **초당 최대 1회만 피격된다.**
  - → 오라/필드·오빗/퍼리미터가 "파고들어 몸빵"해도 다중 히트 즉사가 구조적으로 불가능. 12패밀리 중 2개의 설계 의도가 이 한 줄로 성립한다.
- **실드 흡수 시에도 i-frame이 동일하게 발동**하고 데미지는 0. (실드 스택·무피격 판정은 각 섹션 소관.)
- **폭탄 발동 중 무적**: `bomb.iFrames = 1.5` 게임초 (슈팅 전통. 폭탄 = 살아남는 카드).
- i-frame 길이는 **배속과 무관하게 게임시간 기준** — 난이도=배속 원칙 유지(배속이 곧 실제 반응 여유를 줄이는 것이 난이도의 정의).

### 1.5 몸통 충돌 — **데미지 있음, 단 i-frame 공유 (확정)**

| 대상 | 키 | 값 |
|---|---|---|
| 일반 잡몹 | `enemy.contactDamageDefault` | 8 |
| 엘리트 | `enemy.contactDamageElite` | 12 |
| 중간보스/보스(부위 포함) | `enemy.contactDamageBoss` | 16 |

- 적 스키마의 `contactDamage` 필드로 개체별 오버라이드 가능(기본값은 위 티어).
- **왜 0이 아닌가**: 접촉이 무해하면 "적 위에 얹혀서 오라로 갈기기"가 무위험 최적해가 되어 *의미 있는 이동* 기둥이 소멸한다.
- **왜 즉사가 아닌가(194X식 폐기)**: 오빗/오라의 "파고들기" 자세가 성립 불가해진다.
- **결론**: 접촉 = **초당 최대 1회, 탄보다 낮은 데미지** → *파고들기는 감수할 만한 비용*. 두 패밀리의 의도가 정확히 살아난다.

### 1.6 런 시작 상태 (플레이어 한정)

| 항목 | 값 |
|---|---|
| HP | `hpMax` 만땅 (100) |
| 방어력 | 0 |
| 스탠스 | **노말 (Q)** — `stance.startElement = "normal"` |
| 속성 투자 | 전부 0 |
| 폭탄 스톡 | `bomb.stockStart = 1` (상한 `bomb.stockMax = 3`) |
| 실드 / 시간 토큰 / 리롤 | 0 |
| 코인 | 0 |

*(시작 무기·기체 선택 여부는 「런 시작/드래프트」 섹션 소관.)*

### 1.7 상점 스탯 항목의 상한 (무한 스택 금지)

| 항목 | 키 | 1스택 | 최대 스택 | 최대 도달치 |
|---|---|---|---|---|
| 방어력 ↑ | `shop.defenseStep` / `shop.defenseMaxStacks` | +2 | 4 | 방어력 8 |
| 최대 HP ↑ | `shop.hpMaxStep` / `shop.hpMaxMaxStacks` | +10 (구매 즉시 같은 양 회복) | 4 | 140 |
| 이동 속도 ↑ | `shop.moveSpeedStep` / `shop.moveSpeedMaxStacks` | +6% (기본치에 가산) | 4 | 347.2 |

→ 상한이 있으므로 "코인으로 스탯을 무한히 사서 빌드를 만든다"는 대원칙 우회가 구조적으로 차단된다.

## 2. 조작 (양손 키보드 전용)

> **플랫폼 확정**: 데스크톱 **키보드 전용**. 마우스 미사용(슬롯 재정렬의 드래그 안은 **폐기**). 모바일/터치 **미지원**.
> 키는 `event.code`(물리 키 위치)로 읽는다 → 비-QWERTY 레이아웃에서도 손 위치는 동일. **키 리맵 없음**(4주 예산).

### 2.1 키 맵 (전체 — 미할당 키 없음)

**왼손 (홈: 새끼 Q · 약지 W · 중지 E · 검지 R · 엄지 Space)**

| 키 | 인게임 | 드래프트/상점 화면 | 일시정지 화면 |
|---|---|---|---|
| **Q** | 스탠스 → **노말** | (비활성) | — |
| **W** | 스탠스 → **불** | (비활성) | — |
| **E** | 스탠스 → **물** | (비활성) | — |
| **R** | 스탠스 → **풀** | (비활성) | — |
| **Space** | **폭탄 발동** | 슬롯 재정렬: 집기/놓기 | — |
| **Shift(좌)** | **시간 연장 토큰 발동** | — | — |
| **F** | — | **리롤** (스톡 1 소모) | — |
| **1 / 2 / 3** | (비활성) | 드래프트 카드 즉시 선택 | — |

**오른손 (방향키)**

| 키 | 인게임 | 메뉴/화면 |
|---|---|---|
| **← ↑ → ↓** | 이동 | 커서 이동 / 슬롯 커서 |
| **Enter** | — | 확정 |
| **Esc** | **일시정지** | 화면 닫기/취소 |

- **WASD 폐기** — `W`가 스탠스 키와 물리적으로 충돌. 이동은 방향키 확정.
- **`1234` 스탠스 안 폐기** — ① 숫자열은 홈 포지션에서 손을 위로 뻗어야 함(패닉 순간 오타) ② 드래프트 3택 선택(1/2/3)과 정면 충돌. **QWER 확정.**
- **모드별 배타**: 인게임에서 1/2/3은 죽은 키, 드래프트 화면에서 QWER은 죽은 키. 오조작 원천 차단.

### 2.2 QWER 할당과 그 근거

| 키 | 속성 | 손가락 |
|---|---|---|
| **Q** | **노말 (무속성)** | 새끼 |
| **W** | **불** | 약지 |
| **E** | **물** | 중지 |
| **R** | **풀** | 검지 |

1. **Q = 노말 = 홈/기본.** 런 시작 스탠스이자 "모르겠으면 Q" 안전 바닥. 가장 바깥 키 = 기본값이라는 관습과 일치.
2. **★ 상성 니모닉: "적 속성 키의 *오른쪽 이웃*이 정답 키. R 다음은 W로 순환."**

   | 적 속성 | 적 속성의 키 | 오른쪽 이웃 | = 눌러야 할 키 | 검증 |
   |---|---|---|---|---|
   | 불 | W | E | **E (물)** | 물 → 불 ×2 ✓ |
   | 물 | E | R | **R (풀)** | 풀 → 물 ×2 ✓ |
   | 풀 | R | (순환) W | **W (불)** | 불 → 풀 ×2 ✓ |

   상성 사이클 물>불>풀>물이 **키 배열의 물리적 순환에 그대로 사상된다.** 플레이어는 상성표를 외울 필요 없이 손가락으로 외운다 = 인지부하 최소화(기둥) + 튜토리얼 한 줄로 설명 가능.
3. **QWER 행은 학습 비용 0** — MOBA/RTS의 스킬 행. 왼손 4손가락이 이동 없이 얹힌다.
4. **Space = 폭탄 (엄지)** — 패닉 순간의 액션이므로 **손이 전혀 움직이지 않는 키**여야 한다. 슈팅 전통과도 일치.
5. **Shift = 시간 연장 토큰 (새끼, Q에서 살짝 아래)** — 보스전당 최대 1회의 **신중한 결정**이므로 손가락이 *약간 움직여야 하는* 키가 오히려 옳다(오발 방지). 폭탄(무이동)과 토큰(소이동)의 비대칭은 의도된 설계.

### 2.3 스탠스 규칙

| 규칙 | 값 |
|---|---|
| 전환 쿨다운 | **0** (`stance.switchCooldown = 0`) |
| 전환 딜레이/캐스팅 | 없음. 다음 틱부터 즉시 적용 |
| **런 전체 유지** | `stance.persistAcrossStages = true` — 스테이지 전환·보스 진입·드래프트·상점·컨티뉴 후에도 **유지**. 머슬메모리를 위해 키를 고정한 취지상 리셋은 금지 |
| 부여 재계산 | 속성 레벨 상승 / 새 무기 획득 / 슬롯 재정렬 시 **즉시(다음 틱) 재계산**. 드래프트 화면에서 `불 +1`을 고르면 그 화면에서 바로 부여 프리뷰가 갱신됨 |
| 사망/컨티뉴 | 스탠스 유지 |

*(이미 발사된 탄·설치물의 속성 귀속 = 「무기 × 스탠스」 섹션 소관.)*

### 2.4 슬롯 재정렬 — **시점 제한 (★ 게이트 보호)**

- **조작**: `←/→` 슬롯 커서 이동 → `Space` 집기 → `←/→`로 옮김 → `Space` 놓기(스왑). `Esc` 종료. **마우스 불필요.**
- **가능 시점 (확정)**:

  | 시점 | 재정렬 |
  |---|---|
  | 잡몹 페이즈 중 뜬 레벨업 드래프트 (선택 확정 후) | **가능** |
  | 스테이지 사이 (상점 화면 내 '빌드' 탭) | **가능** |
  | **보스 페이즈 중 뜬 레벨업 드래프트** | **불가** (카드 선택만) |
  | **일시정지 화면** | **불가** (조회 전용) |

- **왜**: 속성 스탠스 시스템의 유일한 게이트는 "투자 레벨 N + 슬롯 순서"다. 보스 부위마다 정지→재정렬→재개가 가능하면 그 게이트가 통째로 우회되고(=투자가 무의미), 동시에 사용자가 명시적으로 싫어하는 **체스식 정지 최적화**가 된다. 재정렬은 **전투 밖의 계획**이지 전투 중의 조작이 아니다.
- 새 무기는 **가장 앞의 빈 슬롯에 자동 배치**되며, 배치 직후 그 드래프트 화면에서 바로 재정렬할 수 있다(획득 순서가 몰래 부여 우선순위가 되는 사고 방지).

### 2.5 일시정지 · 포커스

| 규칙 | 값 |
|---|---|
| 키 | `Esc` |
| 게임 클럭 | **정지** (보스 타이머 포함) |
| 화면 내용 | **조회 전용** — 현재 빌드(무기 4슬롯 + 부여 상태), 속성 투자, **상성표**, 키맵. **어떤 상태 변경도 불가** (재정렬·구매·사용 전부 금지) |
| 재개 | `Esc` → **1.5초 카운트다운** 후 재개 (`pauseResumeCountdown = 1.5`) |
| 브라우저 포커스 상실 (`blur`) | **자동 일시정지** — 세이브 없는 30분 런이 알트탭으로 죽는 것은 회복 불가능한 손실 |

- 상태 변경이 불가하므로 일시정지는 이득을 만들지 못한다 → 무제한 허용해도 안전.
- 상성표를 일시정지에 상시 비치 = 온보딩 비용 0 (심사자가 언제든 확인 가능).

### 2.6 입력 처리 규칙

- **매 고정 틱마다 키 상태를 폴링**한다 (keydown 이벤트 큐 아님). 고정 타임스텝·헤드리스 시뮬 재현성의 전제.
- **동시 반대 키(←+→)**: **마지막에 눌린 키 우선** (SOCD last-input-priority). 둘 다 누른 채 하나를 떼면 남은 쪽으로 즉시 이동.
- 스탠스 키를 여러 개 동시에 누르면 **마지막에 눌린 키** 채택.
- 게임이 사용하는 모든 키는 `preventDefault` (특히 `Space` 스크롤, `Tab`, 방향키 스크롤).

## 3. HUD

### 3.1 화면 분할

```
┌──────────────── 논리 캔버스 1280 × 720 ────────────────────────┐
│ 좌 패널 360        │  플레이필드 560×720  │     우 패널 360     │
│                    │ ┌──────────────────┐ │                     │
│ ── 스탠스/투자 ──  │ │▓  2:47   ×1.0   ▓│ │ SCORE   1,284,300   │
│  Q 노말 ◆ ●●●●   │ │  (보스 타이머)    │ │ BEST    2,010,500   │
│ ▶W 불   ▲ ●●○○   │ │                  │ │ ─────────────────── │
│  E 물   ● ●○○○   │ │                  │ │ ◎ 코인      137     │

│ ── 무기 슬롯 ──    │ │                  │ │ 폭탄  ●●○           │

│ ○ ○ ○ ○ ○         │ │ ▬▬▬▬▬▭ Lv.23    │ │ 화산 ▲불 70%        │

```

- **분할 원칙**: *눈이 항상 가 있는 곳(기체)* → *가끔 흘끗 보는 곳(플레이필드 띠)* → *드래프트/보스 진입에만 보는 곳(패널)* 의 3계층.
- 색·글리프는 「속성 시각 언어」 섹션의 팔레트·글리프를 **그대로** 사용한다. **하나의 색 = 하나의 속성**이 HUD·기체·탄·카드 전부에서 동일.
- **HUD 원소 순서는 키 순서(Q노말 → W불 → E물 → R풀)를 모든 화면에서 강제한다.** 상성표·드래프트 카드·패널 전부 동일 순서. (기존 문서의 "물·불·풀" 순서 표기는 이 순서로 통일한다.)

### 3.2 ★ 기체 부착 HUD — 스탠스와 부여를 눈이 있는 곳에서 읽는다

이 게임에서 가장 새로운 시스템(= 슬롯 순서 부여)은 **패널에 두면 존재하지 않는 것과 같다.** 그래서 3개를 기체에 붙인다.

| 요소 | 규격 |
|---|---|
| **① 히트박스 코어 링** | 흰 코어(r=4) + **현재 스탠스 색 링**(r=6, 2px). 스탠스를 바꾸면 이 링이 즉시 바뀐다 = 회피하며 보는 스탠스 표시기 |
| **② 임뷰 칩 스트립 (★핵심)** | 기체 **아래 10px**에 4칸 칩(각 5×4px, 간격 2px). **왼쪽부터 슬롯 1~4.** 부여된 슬롯 = 스탠스 색 + 글리프, 미부여 = 회색 무채색. `불+2` 스탠스면 `▪▪▫▫` |
| **③ 상태이상 배지** | 기체 **위 12px**. 둔화/스턴 아이콘 + 잔여 시간 링(원형 게이지). 상태이상은 아이콘 없이는 "조작이 고장난 것"으로 읽힌다 |

- `hud.shipImbueChips = true` / `hud.shipStanceRing = true` (끌 수 있게 하되 기본 ON).
- 스탠스 전환 순간: 기체에서 스탠스 색 링이 1회 확장(120ms, r 14→30, alpha 0.8→0) + 임뷰 칩이 왼쪽부터 순차 점등(칩당 30ms) + 짧은 SFX. **쿨다운이 0이므로 확인 신호는 즉시·명료해야 한다.**

### 3.3 플레이필드 오버레이 (얇은 띠 3개 — 이동 영역 밖)

| 띠 | 위치 | 내용 |
|---|---|---|
| **상단** (y 0~48, 반투명) | 중앙 | **보스 타이머** `2:47` (큰 숫자) + 우측에 **배속 배지** `×1.0` / 잡몹 페이즈 중엔 잔여 시간 |
| **하단 A** (y 672~696) | 전폭 | **HP 세그먼트 바** (5칸, 각 칸 = 20 HP. 부분 감소는 칸 내부가 깎임) + 우측에 `86/100` |
| **하단 B** (y 696~720) | 전폭 | **XP 바** (높이 6px) + 좌측 `Lv.23` |

- **타이머는 게임시간을 표시한다** (항상 3:00 → 0:00). 옆의 `×3.0` 배지가 실제 시간이 압축돼 있음을 알린다. 게임시간이 문서의 불변 규칙이므로 표시도 게임시간으로 통일.
- **빨간불**: `bossTimerRedThreshold = 30` **게임초** 이하 → 숫자 빨강 + 1Hz 펄스 + 플레이필드 상단 테두리 붉은 글로우 + 경고 SFX. **이것이 시간 연장 토큰(Shift)의 발동 신호**다. (토큰의 발동 가능 조건인지 단순 안내인지는 「토큰」 섹션 소관 — HUD는 신호만 책임진다.)
- 적은 y=-40에서 스폰해 반투명 상단 띠를 통과해 내려온다(진입이 미리 보임).

### 3.4 좌 패널 (360px) — 빌드

**① 스탠스 / 속성 투자 블록** (상단, 최우선)

```
▶ W  불  ▲   ●●○○      ← 현재 스탠스 행: 좌측 ▶ 마커 + 행 배경 하이라이트
  E  물  ●   ●○○○
```
- **4행 고정 순서**: `Q 노말` / `W 불` / `E 물` / `R 풀`. 각 행 = **키캡 + 속성명 + 글리프 + 색 + 투자 레벨 핍(●=투자, ○=미투자, 4칸)**.
- **노말 행의 핍은 표시하지 않고 `— (항상 전부 ×1)` 로 표기**한다. 노말은 투자 축이 아니다(다음 항 참조).
- 현재 스탠스 행만 하이라이트 → 4개 투자 레벨과 현재 스탠스를 **한 눈에 동시 판독**.

> **노말 스탠스(Q)가 하는 일 (확정)**: 노말은 **투자 대상이 아니다**(드래프트에 `노말 +1` 카드 없음). Q는 **"전 슬롯 강제 무속성 = 전부 ×1"** 을 **명시적으로 선언**하는 키다. 기계적으로는 미투자 속성 스탠스와 동일하지만, **의도의 표명**이라는 UX 역할이 있다: ① ×0.5 사고 방지 — 잘못된 스탠스로 들어가는 것보다 Q가 항상 안전 ② 노말 부위/노말 적 앞에서 "여긴 상성이 없다"를 스스로 확인하는 리셋 버튼 ③ 런 시작 기본값. Q를 없애고 3키로 줄이면 "지금 뭘 켜뒀더라"의 불안이 상시화된다. **4키 유지.**

**② 무기 슬롯 4칸** (세로 스택, 위=슬롯1)

각 행: `순번 · 아이콘 · 이름 · Lv?/MAX/진화 · **부여 배지**`
- **부여 배지**가 이 패널의 존재 이유: 현재 스탠스에서 그 슬롯이 부여받았으면 **스탠스 색 칩 + 글리프**, 아니면 **회색 `무`**. 임뷰 칩 스트립(3.2)과 항상 1:1 대응.
- 슬롯 순번은 **부여 우선순위**이므로 숫자를 크게. 재정렬 화면에서는 이 리스트가 그대로 커서 UI가 된다.
- 진화한 무기는 슬롯 테두리를 금색으로.

**③ 패시브 5칸** (하단, 작은 아이콘 그리드)

### 3.5 우 패널 (360px) — 자원 / 상황

| 블록 | 내용 |
|---|---|
| 점수 | `SCORE` (현재) / `BEST` (해당 난이도 로컬 최고, 갱신 중이면 금색) |
| 코인 | 아이콘 + 수량. **남은 코인 = 점수**이므로 상시 노출 |
| 소지품 | **폭탄** ●●○ (최대 3) / **실드** ●○ / **시간 토큰** ● / **리롤** ●● — 전부 **핍(도트) 표기**(숫자 아님, 한눈 판독) + 각 항목에 **발동 키캡**을 작게 병기(`폭탄 [Space]`, `토큰 [Shift]`, `리롤 [F]`) = 상시 튜토리얼 |
| 스테이지 | `STAGE 3 / 6` + **테마명 + 테마 속성 글리프/색 + 비율** (`화산 ▲불 70%`) → 스테이지 내내 유지(고지 배너가 사라져도 정보가 남음) |
| 난이도 | `NORMAL ×1.0` |

### 3.6 HUD 공통 규칙

- **HUD는 플레이필드 rect(560×720) 밖에 그린다.** 예외는 3.3의 띠 3개(반투명, 이동 영역 밖)와 3.2의 기체 부착 요소뿐.
- **HUD는 적 탄보다 아래에 그리지 않는다** — 기체 부착 요소는 적 탄 **아래** z-order(탄 가독성 우선), 패널·띠는 플레이필드와 겹치지 않으므로 무관.
- **패널은 정적이다** — 매 프레임 다시 그리지 않고 **값이 바뀔 때만** 오프스크린 캔버스에 갱신 후 blit. ×3 배속에서 CPU 예산 보호.
- 텍스트는 전부 **1px 어두운 아웃라인**(움직이는 배경 위 가독성). 최소 글자 크기 12px.
- 패널 배경은 **무채색 저채도**(속성 4색·적 탄 색과 충돌 금지).

## 5. 이 섹션이 닫은 설계 구멍 (요약)

| 구멍 | 결론 |
|---|---|
| 체력 모델 | 수치형 정수 HP + 세그먼트 표시. 탄마다 데미지 상이. 방어력 = 정액 감산 + 25% 하한 |
| 무적시간 | **존재.** 1.0 게임초, **모든 피해원이 공유** → 오라/오빗 다중 히트 즉사 불가 |
| 몸통 충돌 | 데미지 있음(잡몹 8) + i-frame 공유 + 밀림 없음 → "파고들기"가 감수할 만한 비용으로 성립 |
| 잔기 | **없음** |
| 그레이즈 | **없음** (트위치 유도 → 기둥 위반) |
| 이동 모델 | 관성 0(즉시), 대각선 정규화 필수, 스무딩 항은 존재하되 기본값 0 |
| 히트박스 가시성 | **상시 표시**. 코어 도트 + 스탠스 색 링 |
| 키 확정 | **QWER 확정**(1234 폐기, WASD 폐기). Q노말/W불/E물/R풀 + **"오른쪽 이웃이 정답" 니모닉** |
| 폭탄/토큰 키 | **Space(엄지, 무이동) / Shift(새끼, 소이동)** — 패닉 vs 신중의 의도적 비대칭 |
