- **§13.2-②의 「`random`의 명목 ST ≥ 72.6% of `dpsRef` ⟺ generalist baseline 대비 ≥ 86%」가 재현되지 않는다** — 저작 계약 `draftPolicyDpsRatio(random) ≥ 0.86`의 근거 수치다. 내가 정본의 인쇄값으로 역산하면: s5에서 `T_random(run) ≤ 166.1` ⟺ `T_bal(probe,5)/q ≤ 166.1` ⟺ `133.3/q ≤ 166.1` ⟺ **q ≥ 0.803**(72.6%가 아니다). s3(razor 스테이지)로 재도 `135.0/q ≤ 166.1` → **q ≥ 0.813**. 그리고 72.6 ÷ 83 = 87.5%인데 정본은 86%라 쓴다 — 두 수끼리도 어긋난다. 어느 자연스러운 읽기로도 0.726이 나오지 않으므로 **저작 계약의 임계값이 유도 불명**이다. 게이트 판정은 이 계약이 사후 측정 대상이라 불변이나, **밸런서·시뮬 구현자가 「86%를 어디서 잰 수인가」를 알 수 없다.**
- **§13.2-⑪의 `survivalFirst` 시뮬레이션 서사가 자기 가격표와 어긋난다** — 정본: 「v1 = 0개(**27 < `reroll` 30**) → v2 = 1개(`potion` 40) → **v3 = 1개(`shield` 60)** → **v4 = 1~2개** → v5 = 1~2개 → 중앙값 1.0」. 내가 정본의 가격 공식·`survivalFirst` 정의(`potion`→`shield`→`defense`→`maxhp`)·수입 사다리 [27, 58, 93, 132, 175]로 재계산: **v1 = 0**(27 < potion 40 — reroll은 생존 항목이 아니라 근거가 틀렸다) / v2 = 1(potion 40, 잔액 18) / **v3 = 1이나 `shield` 60이 아니라 `defense` 45**(잔액 53 < 60) / **v4 = 0**(잔액 47 < potion 60 · shield 60 · defense 66 · maxhp 55) / v5 = 1(potion 60) → [0,1,1,0,1] → **중앙값 1.0 ✔ 판정 불변**. 서사의 3곳이 틀렸으나 결론은 같다. 다만 이 게이트는 **하한에 정확히 붙는 razor**이므로(정본도 인정) 서사가 틀린 채로 두면 실패 시 진단이 어긋난다.
